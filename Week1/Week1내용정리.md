# 📘 머신러닝 소개

---

## 1.1 머신러닝이란?

- **머신러닝**: 데이터에서 학습하도록 컴퓨터를 프로그래밍하는 과학  
- **훈련 세트 (training set)**: 시스템이 학습하는 데 사용하는 샘플  
- **훈련 사례 (training instance) / 샘플 (sample)**: 각 훈련 데이터  
- **모델 (model)**: 머신러닝 시스템에서 학습하고 예측을 만드는 부분  

---

## 1.2 왜 머신러닝을 사용하나?

### ✔ 머신러닝의 강점 분야
- 기존 솔루션으로는 많은 수동 조정과 규칙이 필요한 문제  
- 전통적인 방식으로는 해결 방법이 없는 복잡한 문제  
- 유동적인 환경  
- 복잡한 문제와 대량의 데이터에서 인사이트 얻기  

- **데이터 마이닝**: 대용량 데이터를 분석해 숨겨진 패턴을 발견  

---

## 1.3 애플리케이션 사례

- 생산 라인에서 제품 이미지를 분석해 자동으로 분류하기  
- 뇌를 스캔하여 종양 진단하기  
- 자동으로 뉴스 기사 분류하기  

---

## 1.4 머신러닝 시스템의 종류

### 1.4.1 훈련 지도 방식

#### 📌 지도학습 (Supervised Learning)
- 훈련 데이터에 **레이블(정답)** 포함  
  - 분류 (Classification)  
  - 회귀 (Regression)  

#### 📌 비지도학습 (Unsupervised Learning)
- 훈련 데이터에 **레이블 없음**
  - 계층군집 (Hierarchical Clustering): 그룹을 더 작은 그룹으로 세분화  
  - 시각화 (Visualization): 고차원 데이터를 도식화  
  - 차원축소 (Dimensionality Reduction): 정보 손실을 최소화하며 데이터 축소  
  - 이상치 탐지 (Outlier Detection): 비정상 값 제거  
  - 특이치 탐지 (Novelty Detection): 새로운 이상 패턴 탐지  
  - 연관 규칙 학습 (Association Rule Learning): 특성 간 관계 발견  

#### 📌 준지도 학습 (Semi-supervised Learning)
- 일부만 레이블이 있는 데이터 사용  

#### 📌 자기 지도 학습 (Self-supervised Learning)
- 레이블 없는 데이터로부터 레이블 생성  

#### 📌 강화학습 (Reinforcement Learning)
- **에이전트**: 환경을 관찰하고 행동 수행  
- **보상/벌점**: 행동 결과  
- **정책 (Policy)**: 어떤 행동을 선택할지 정의  

---

### 1.4.2 배치 학습과 온라인 학습

- 배치학습 (Batch Learning)  
- 온라인 학습 (Online Learning)  
- 외부 메모리 학습 (Out-of-core Learning)  

#### 📌 온라인 학습 주요 개념
- **학습률 (Learning Rate)**: 데이터 변화에 얼마나 빠르게 적응할지 결정  

#### ⚠️ 온라인 학습의 문제점
- 나쁜 데이터 입력 시 성능 감소  
- 성능 모니터링 필수  
- 이상 데이터 감지 필요  

---

### 1.4.3 사례 기반 vs 모델 기반 학습

#### 📌 사례 기반 학습 (Instance-based Learning)
- 훈련 데이터를 기억  
- 유사도 기반으로 예측  

#### 📌 모델 기반 학습 (Model-based Learning)
- 데이터로 모델 생성 후 예측  

---

## 🔄 머신러닝 프로젝트 과정

1. 데이터 분석  
2. 모델 선택  
3. 모델 훈련  
   - 비용 함수를 최소화하는 파라미터 학습  
4. 예측 (추론, Inference)
