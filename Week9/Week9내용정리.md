## 6.1 결정 트리 학습과 시각화

### 결정 트리 (Decision Tree)란?

- **분류**(classification)와 **회귀**(regression) 작업, 그리고 **다중 출력**(multi-output) 작업까지 가능한 다목적 머신러닝 알고리즘
- 매우 복잡한 데이터셋도 학습할 수 있는 **강력한 알고리즘**
- **랜덤 포레스트**(Random Forest)의 기본 구성 요소가 됨 (랜덤 포레스트는 여러 결정 트리를 앙상블한 모델)

### 학습 흐름 (붓꽃 데이터셋 예시)

붓꽃(iris) 데이터셋에서 꽃잎 길이(petal length)와 꽃잎 너비(petal width) 두 특성만 사용하여 `DecisionTreeClassifier`를 `max_depth=2`로 훈련시킨다.

### 시각화 방법

1. `sklearn.tree.export_graphviz()` 함수로 훈련된 트리를 `.dot` 파일로 출력
2. `graphviz.Source.from_file()`을 통해 주피터 노트북에 로드하여 시각화
3. 필요 시 `dot -Tpng` 명령으로 `.png` 이미지 파일로 변환 가능

## 6.2 예측 (Prediction)

### 그림 6-1 붓꽃 결정 트리의 구조

훈련된 결정 트리는 다음과 같은 구조로 구성된다.

#### 노드(Node)의 종류

| 노드 종류 | 설명 |
|---|---|
| **루트 노드 (root node)** | 깊이(depth)가 0인 맨 꼭대기 노드. 첫 번째 분할 조건을 검사. 예: "꽃잎 길이가 2.45cm보다 짧은가?" |
| **분할 노드 (split node)** | 자식 노드를 가지며 분할 조건을 가진 노드 |
| **리프 노드 (leaf node)** | 자식 노드를 가지지 않는 끝단 노드. 클래스 예측을 수행하는 곳 |

#### 각 노드가 가지는 속성

- **samples**: 그 노드에 적용된 훈련 샘플의 개수
- **value**: 노드에서 각 클래스에 속한 훈련 샘플의 분포 (예: `[50, 0, 0]` → setosa 50개, versicolor 0개, virginica 0개)
- **gini**: 그 노드의 **지니 불순도**(Gini impurity)
- **class**: 그 노드에서 가장 비율이 높은 클래스 (예측값)

### 식 6-1: 지니 불순도 (Gini Impurity)

$$G_i = 1 - \sum_{k=1}^{n} p_{i,k}^{\,2}$$

- $G_i$ : $i$번째 노드의 지니 불순도
- $p_{i,k}$ : $i$번째 노드에 있는 훈련 샘플 중 클래스 $k$에 속한 샘플의 비율
- $n$ : 클래스 개수

#### 지니 불순도의 의미
- **불순도가 0**: 그 노드의 모든 샘플이 같은 클래스 → 완벽하게 순수(pure)한 노드
- **불순도가 클수록**: 노드 안에 여러 클래스가 섞여 있음

#### 계산 예시 (그림 6-1, 깊이 2의 왼쪽 노드)

해당 노드는 samples=54, value=[0, 49, 5]이다.

$$G = 1 - \left(\frac{0}{54}\right)^{2} - \left(\frac{49}{54}\right)^{2} - \left(\frac{5}{54}\right)^{2} \approx 0.168$$

→ 비교적 순수한 편 (49/54가 versicolor)

### 예측 과정

새 샘플이 들어오면 루트 노드부터 시작해서 각 분할 노드의 조건을 검사하며 트리를 따라 내려가고, 도달한 리프 노드의 클래스를 예측값으로 반환한다.

### 그림 6-2: 결정 트리의 결정 경계 (Decision Boundary)

붓꽃 데이터의 두 특성(꽃잎 길이, 꽃잎 너비)을 평면에 그리고 결정 경계를 시각화한 결과다.

- **굵은 수직선**: 루트 노드의 결정 경계 → 꽃잎 길이 = 2.45cm
  - 왼쪽 영역: 모두 Iris setosa (순수, 더 분할 안 함)
- **굵은 가로 파선**: 깊이 1의 오른쪽 노드 결정 경계 → 꽃잎 너비 = 1.75cm
  - 위쪽: 주로 virginica, 아래쪽: 주로 versicolor
- **두 개의 수직 점선**: `max_depth=3`일 때 깊이 2의 두 노드가 추가로 만드는 결정 경계

→ 결정 트리의 결정 경계는 **항상 축에 수직인 직선들의 조합**(계단 모양)이다.

### 모델 해석: 화이트박스 vs 블랙박스

#### 화이트박스(White Box) 모델
- 결정 트리처럼 결정 방식을 **직관적으로 이해**할 수 있는 모델
- 사람이 트리를 따라가며 직접 분류 결과를 추적 가능
- "왜 그런 예측을 했는지" 설명이 쉬움

#### 블랙박스(Black Box) 모델
- 랜덤 포레스트, 신경망 등의 알고리즘
- 성능이 뛰어나고 예측을 만드는 **연산 과정 자체는 확인 가능**
- 그러나 **"왜 그런 예측을 만드는지"** 설명하기 어려움

#### 해석 가능한 ML (Interpretable ML)
- 사람이 이해할 수 있는 방식으로 모델의 결정을 설명할 수 있는 ML 시스템을 만드는 분야
- 시스템이 **불공정한 결정**을 내리지 않도록 하는 등의 측면에서 매우 중요
- 결정 트리는 필요하다면 수동으로도 따라 해볼 수 있는 **간단하고 명확한 분류 방법**을 사용한다는 점에서 해석 가능성이 매우 높음

## 6.3 클래스 확률 추정 (Class Probability Estimation)

결정 트리는 한 샘플이 특정 클래스 $k$에 속할 확률도 추정할 수 있다.

### 추정 방식

1. 샘플이 도달하는 **리프 노드**를 찾기 위해 트리를 탐색
2. 그 리프 노드의 `value`를 보고, **클래스 $k$의 훈련 샘플의 비율**을 반환

### 예시: 길이 5cm, 너비 1.5cm 인 꽃잎

이 샘플은 깊이 2의 왼쪽 리프 노드(samples=54, value=[0, 49, 5])에 도달한다. 따라서 결정 트리가 출력하는 확률은:

| 클래스 | 확률 |
|---|---|
| Setosa | 0 / 54 = **0%** |
| Versicolor | 49 / 54 ≈ **90.7%** |
| Virginica | 5 / 54 ≈ **9.3%** |

→ 가장 높은 확률을 가진 **Versicolor (클래스 1)** 가 예측값.

### 중요한 특성

같은 리프 노드에 도달하는 모든 샘플은 **같은 추정 확률**을 갖는다. 예를 들어 길이 6cm, 너비 1.5cm인 꽃잎도 같은 리프 노드에 도달하므로 확률이 동일하다.

→ 결정 경계 안에서는 확률이 일정하고, 경계를 넘어갈 때 갑자기 바뀌는 **계단 형태**의 확률 분포를 갖는다.

## 6.4 CART 훈련 알고리즘

사이킷런이 결정 트리를 훈련시킬 때 사용하는 알고리즘은 **CART(Classification And Regression Tree)** 이다.

### 알고리즘 동작 방식

1. 훈련 세트를 하나의 특성 $k$ 와 그에 대한 임곗값 $t_k$ 를 사용해 두 개의 서브셋으로 나눈다.
   - 예) "꽃잎의 길이 ≤ 2.45cm" 인지 여부로 분할
2. $k$ 와 $t_k$ 는 **가장 순수한 서브셋**을 만들 수 있는 $(k, t_k)$ 쌍을 찾아 결정 (비용 함수 최소화)
3. 성공적으로 둘로 나눈 다음, 각 서브셋에 대해 **같은 방식으로 재귀적으로** 분할
4. 다음 조건 중 하나에 도달하면 분할을 멈춤:
   - `max_depth`에 도달
   - 불순도를 줄이는 분할을 더 찾을 수 없음
   - 그 외 규제 매개변수 조건 (min_samples_split 등)

### 식 6-2: 분류에 대한 CART 비용 함수

$$J(k, t_k) = \frac{m_{\text{left}}}{m}\, G_{\text{left}} + \frac{m_{\text{right}}}{m}\, G_{\text{right}}$$

- $G_{\text{left/right}}$ : 왼쪽 / 오른쪽 서브셋의 불순도(지니 불순도 등)
- $m_{\text{left/right}}$ : 왼쪽 / 오른쪽 서브셋의 샘플 수
- $m$ : 부모 노드의 전체 샘플 수

→ 두 서브셋의 **샘플 수에 가중평균한 불순도**를 최소화하는 $(k, t_k)$ 를 찾는 것이 목표.

### CART 알고리즘의 특성: 탐욕적(Greedy) 알고리즘

- 각 단계에서 **국소적으로 최적인 분할**(현재 단계에서의 최선)을 선택
- 전역적으로 최적의 트리를 보장하지는 않음 (전역 최적 트리를 찾는 문제는 NP-완전)
- 그러나 일반적으로 충분히 좋은 성능을 보이는 트리를 생성함

## 6.5 계산 복잡도 (Computational Complexity)

### 예측 시 복잡도

- 결정 트리는 루트 노드에서 리프 노드까지 탐색하면서 예측
- 일반적으로 트리는 **균형 잡힌 이진 트리**에 가깝게 만들어지므로:
  - 깊이 $d$ 에서의 리프 노드 개수는 약 $2^{d}$
  - 리프 노드 개수가 훈련 데이터 수 $m$ 이라면 트리의 깊이는 약 $\log_2(m)$
- 각 노드에서는 **하나의 특성값만** 확인하면 됨

따라서 예측의 전체 복잡도는 **특성 개수와 무관하게**:

$$O(\log_2(m))$$

→ 큰 훈련 세트를 다룰 때에도 **예측 속도가 매우 빠르다**는 것이 결정 트리의 큰 장점.

### 훈련 시 복잡도

훈련 알고리즘은 **각 노드에서** 모든 훈련 샘플의 **모든 특성**(또는 `max_features`로 제한된 일부)을 비교하며 최적의 분할을 찾는다.

각 노드에서 모든 샘플의 모든 특성을 비교한다면:

$$O(n \times m \log_2(m))$$

- $n$ : 특성의 개수
- $m$ : 훈련 샘플의 개수

## 6.6 지니 불순도 또는 엔트로피?

`DecisionTreeClassifier`는 기본적으로 **지니 불순도**를 사용하지만, `criterion="entropy"`로 지정하면 **엔트로피**를 사용할 수도 있다.

### 식 6-3: 엔트로피 (Entropy)

$$H_i = -\sum_{\substack{k=1 \\ p_{i,k}\neq 0}}^{n} p_{i,k} \log_2(p_{i,k})$$

- $H_i$ : $i$번째 노드의 엔트로피
- $p_{i,k}$ : $i$번째 노드에 있는 훈련 샘플 중 클래스 $k$의 비율
- 비율이 0인 항은 합산에서 제외 (로그 연산을 피하기 위해)

### 엔트로피의 의미

엔트로피는 정보이론에서 유래한 개념으로, 어떤 집합의 "무질서도" 또는 "불확실성"을 측정한다.
- 한 클래스만 있는 노드(완전 순수): 엔트로피 = 0
- 클래스가 균등하게 섞일수록 엔트로피가 커짐

### 계산 예시 (깊이 2의 왼쪽 노드)

samples=54, value=[0, 49, 5]:

$$H = -\frac{49}{54}\log_{2}\!\left(\frac{49}{54}\right) - \frac{5}{54}\log_{2}\!\left(\frac{5}{54}\right) \approx 0.445$$

(클래스 0의 비율이 0이므로 그 항은 제외)

### 지니 vs 엔트로피, 어떤 걸 쓸까?

| 항목 | 지니 불순도 | 엔트로피 |
|---|---|---|
| 계산 속도 | **조금 더 빠름** (로그 계산이 없음) | 상대적으로 느림 |
| 결과 트리 | 가장 빈도 높은 클래스를 한쪽 가지(branch)로 **고립시키는 경향** | 조금 더 **균형 잡힌 트리** 생성 |
| 일반적 사용 | 기본값으로 좋음 | 큰 차이 없음 |

대부분의 경우 두 방법은 **비슷한 트리**를 만들어 내며, 차이가 큰 경우에 두 결과를 비교해보면 된다.

## 6.7 규제 매개변수 (Regularization Hyperparameters)

### 비파라미터 모델 vs 파라미터 모델

#### 비파라미터 모델 (Nonparametric Model)
- **결정 트리**는 비파라미터 모델
- 훈련되기 전에 파라미터 수가 결정되지 않음
- 모델 구조가 데이터에 맞춰서 자유롭게 변형됨
- 자유도가 높음 → **과대적합(overfitting)** 위험이 큼

#### 파라미터 모델 (Parametric Model)
- 선형 모델처럼 데이터의 형태를 미리 가정
- 모델 파라미터 수가 미리 정해져 있음
- 자유도가 제한됨 → **과대적합 위험은 줄지만, 과소적합(underfitting) 위험은 큼**

### 결정 트리의 규제: 트리 형태 제한

`DecisionTreeClassifier`에서 트리의 형태를 제한하는 주요 하이퍼파라미터:

| 매개변수 | 의미 |
|---|---|
| `max_depth` | 트리의 **최대 깊이** (가장 대표적인 규제 매개변수) |
| `max_features` | 각 노드에서 분할에 사용할 **특성의 최대 수** |
| `max_leaf_nodes` | **리프 노드의 최대 수** |
| `min_samples_split` | 노드가 분할되기 위해 가져야 하는 **최소 샘플 수** |
| `min_samples_leaf` | 리프 노드가 생성되기 위해 가지고 있어야 할 **최소 샘플 수** |
| `min_weight_fraction_leaf` | `min_samples_leaf`와 같지만, 가중치가 부여된 전체 샘플 수에서의 **비율(fraction)** |

#### 일반적인 규제 방향
- `max_*` 매개변수는 **줄일수록** 모델 규제가 강해짐
- `min_*` 매개변수는 **늘릴수록** 모델 규제가 강해짐

### 그림 6-3: 규제의 효과 (moons 데이터셋 예시)

- **왼쪽 (No restrictions)**: 규제하지 않은 결정 트리
  - 결정 경계가 매우 복잡하고, 노이즈에 과도하게 반응함
  - 명백히 **과대적합** 상태
- **오른쪽 (`min_samples_leaf=5`)**: 규제를 추가한 결정 트리
  - 결정 경계가 더 부드럽고 데이터의 일반적인 패턴을 잘 따름

### 일반화 성능 비교

다른 랜덤 시드로 생성한 테스트 세트(`random_state=43`, 1000개 샘플)에서 두 트리를 평가했을 때:

| 모델 | 정확도 |
|---|---|
| 규제 없음 | 0.898 |
| `min_samples_leaf=5` | **0.92** |

→ 규제를 추가한 모델의 일반화 성능(정확도)이 더 높다.

## 6.8 회귀 (Regression)

결정 트리는 회귀 작업에도 사용 가능하며, `sklearn.tree.DecisionTreeRegressor` 클래스를 사용한다.

### 그림 6-4: 회귀 결정 트리

잡음이 섞인 2차 함수 형태의 데이터셋(`X_quad ~ Uniform(-0.5, 0.5)`, `y = X^2 + noise`)에 `max_depth=2`로 회귀 트리를 훈련시킨 예시.

#### 분류 트리와 다른 점

각 노드의 속성이 분류 트리와 다르다.

| 분류 트리 | 회귀 트리 |
|---|---|
| `gini` (불순도) | `squared_error` (해당 노드 샘플들의 MSE) |
| `value` = 클래스별 샘플 개수 | `value` = 그 노드 샘플들의 **타깃값 평균** (예측값) |
| 예측: 가장 많은 클래스 | 예측: `value` (평균) |

### 그림 6-5: max_depth=2 vs max_depth=3 비교

- `max_depth=2`: 영역이 4개로 분할 → 4개의 일정한 예측값(계단)
- `max_depth=3`: 영역이 더 잘게 분할 → 데이터의 곡선을 더 세밀하게 근사

#### 핵심 원리

- **각 영역의 예측값은 항상 그 영역에 속한 타깃값들의 평균**이다.
- 알고리즘은 **예측값과 가능한 한 많은 샘플이 가깝도록(=MSE가 작도록)** 영역을 분할한다.

### 식 6-4: 회귀를 위한 CART 비용 함수

분류에서는 불순도(Gini, Entropy)를 최소화했지만, 회귀에서는 **MSE를 최소화**하도록 분할한다.

$$J(k, t_k) = \frac{m_{\text{left}}}{m}\,\text{MSE}_{\text{left}} + \frac{m_{\text{right}}}{m}\,\text{MSE}_{\text{right}}$$

여기서:

$$\text{MSE}_{\text{node}} = \frac{1}{m_{\text{node}}}\sum_{i \in \text{node}}(\hat{y}_{\text{node}} - y^{(i)})^{2}$$

$$\hat{y}_{\text{node}} = \frac{1}{m_{\text{node}}}\sum_{i \in \text{node}} y^{(i)}$$

- $\hat{y}_{\text{node}}$ : 그 노드의 예측값 = 노드에 속한 샘플들의 **평균**
- $\text{MSE}_{\text{node}}$ : 그 노드 안 샘플들의 **분산**과 동일한 형태

→ 분류는 "불순도(섞임)" 최소화, 회귀는 "분산(흩어짐)" 최소화.

### 그림 6-6: 회귀에서의 규제

- **왼쪽 (No restrictions)**: 규제 없는 회귀 트리
  - 모든 데이터 포인트를 거의 그대로 따라가는 매우 들쭉날쭉한 예측
  - 명백한 **과대적합**
- **오른쪽 (`min_samples_leaf=10`)**: 규제 있는 회귀 트리
  - 부드럽고 합리적인 계단 형태로 2차 함수를 근사

→ 회귀에서도 분류와 마찬가지로 결정 트리는 **과대적합되기 쉬우므로 규제가 필수**다.

## 6.9 축 방향에 대한 민감성 (Sensitivity to Axis Orientation)

### 결정 트리의 한계

결정 트리의 모든 분할은 **축에 수직(orthogonal to an axis)** 이다. 즉, 결정 경계가 항상 **계단 모양**이다.

이로 인해 다음과 같은 단점이 있다:

- **데이터의 방향(회전)에 매우 민감** : 데이터셋을 단순히 회전시키기만 해도 결정 경계가 완전히 달라짐.

### 그림 6-7: 데이터 회전에 민감한 결정 트리

- **왼쪽**: 두 클래스가 수직선 하나로 깔끔히 분리되는 데이터
  - 트리가 단 하나의 분할 (`x1 = 0`) 으로 완벽히 분리
- **오른쪽**: 같은 데이터를 45° 회전한 경우
  - 같은 패턴인데도 트리가 **계단 모양**의 복잡한 경계를 만들어야 함
  - 일반화 성능이 나빠질 가능성이 높음

### 해결책: PCA를 활용한 데이터 변환

데이터의 스케일을 조정한 다음 **주성분 분석(Principal Component Analysis, PCA)** 을 적용하면:

- 특성 간의 **상관관계를 줄이는** 방향으로 데이터가 회전됨
- 결정 트리가 더 쉽게 **축에 정렬된 분할**을 찾을 수 있게 됨

### 그림 6-8: PCA 회전한 붓꽃 데이터셋의 결정 경계

- 표준화(`StandardScaler`) + PCA 변환을 거친 붓꽃 데이터에 `max_depth=2`로 결정 트리를 훈련
- 결과: PCA로 변환한 좌표축($z_1, z_2$)에서 보면 단 두 개의 수직 분할(`Depth=0`, `Depth=1`) 만으로도 세 클래스가 거의 완벽히 분리됨
- 원본 좌표계에서는 더 복잡한 경계가 필요했을 분류 문제가, 회전 후에는 훨씬 단순해짐

→ 결정 트리를 학습시키기 전에 **전처리(스케일링 + PCA)** 를 하는 것이 도움 될 수 있음.

## 6.10 결정 트리의 분산 문제 (High Variance)

### 결정 트리의 주요 문제: 높은 분산

결정 트리는 **분산이 상당히 큰 모델**이다. 즉:

- 하이퍼파라미터를 조금만 바꿔도 매우 다른 모델이 생성됨
- 데이터를 조금만 변경하거나 (혹은 같은 데이터에서 단지 **다른 random_state** 로 재학습하더라도) 매우 다른 모델이 만들어짐

### 그림 6-9: 같은 데이터, 다른 랜덤 시드 → 다른 모델

`max_depth=2`로 동일한 붓꽃 데이터에 훈련시키되 `random_state`만 바꿨을 때:

- 그림 6-2(원래 모델)와 그림 6-9(`random_state=40`)는 **완전히 다른 결정 경계**를 보여줌
- 같은 알고리즘 + 같은 데이터인데 결과가 다름
  - CART 알고리즘이 동일한 비용을 가진 분할 후보들 중 무작위로 하나를 선택하기 때문

### 해결책: 앙상블 (Ensemble)

여러 결정 트리의 예측을 **평균**(혹은 다수결)하면 분산을 크게 줄일 수 있다.

→ 이 아이디어가 바로 **랜덤 포레스트(Random Forest)** 의 핵심이다.

## 연습문제

다음은 강의 자료의 연습문제와 풀이 핵심이다.

### 1. 백만 개의 샘플을 가진 훈련 세트에서 (규제 없이) 훈련시킨 결정 트리의 깊이는 대략 얼마일까?

균형 잡힌 이진 트리라면 리프 노드 수가 $m$일 때 깊이는 약 $\log_2(m)$이다.

$$\log_2(10^6) \approx 20$$

→ 실제로는 완벽히 균형 잡히지 않으므로 조금 더 깊어질 수 있지만, 대략 **20 정도**.

### 2. 한 노드의 지니 불순도가 보통 그 부모 노드보다 작을까, 클까? 일반적으로 작거나 클까, 아니면 항상 작거나 클까?

- **일반적으로는 작다(낮다).** CART의 비용 함수는 자식 노드들의 **가중평균 불순도**를 최소화하므로, 부모보다 가중평균이 작은 분할만 선택됨.
- **그러나 항상은 아니다.** 어느 한쪽 자식의 불순도는 부모보다 클 수도 있다 (다른 쪽 자식이 더 순수해서 가중평균이 부모보다 작기만 하면 됨).

### 3. 결정 트리가 훈련 세트에 과대적합되었다면 max_depth를 줄이는 것이 좋을까?

→ **그렇다.** `max_depth`를 줄이면 모델이 단순해져 규제 효과가 생기고 과대적합이 완화된다.

### 4. 결정 트리가 훈련 세트에 과소적합되었다면 입력 특성의 스케일을 조정하는 것이 좋을까?

→ **아니다.** 결정 트리는 각 특성에 대해 임곗값을 비교할 뿐, 특성들 간의 거리를 사용하지 않으므로 **특성 스케일에 영향을 받지 않는다.** 따라서 스케일 조정은 과소적합 해결에 도움이 되지 않는다.

### 5. 백만 개 샘플 훈련에 1시간이 걸렸다면, 천만 개 샘플은?

훈련 복잡도는 $O(n \times m \log_2(m))$이다. 같은 $n$에서 $m$이 10배 증가하면:

$$\frac{10m \cdot \log_2(10m)}{m \cdot \log_2(m)} = 10 \cdot \frac{\log_2(10) + \log_2(m)}{\log_2(m)}$$

$m = 10^6$ 일 때 $\log_2(m) \approx 20$, $\log_2(10) \approx 3.32$ 이므로:

$$10 \times \frac{3.32 + 20}{20} \approx 10 \times 1.166 \approx 11.66$$

→ 약 **11.7시간** 정도 걸린다.

### 6. 어떤 훈련 세트에서 결정 트리를 훈련하는 데 1시간이 걸린 경우, 특성의 개수를 두 배로 늘리면?

훈련 복잡도가 $n$에 대해 선형이므로 ($O(n \times m \log_2 m)$), 특성 수가 2배가 되면 시간도 약 **2배** → **2시간**.

### 7. moons 데이터셋에 결정 트리를 훈련시키고 세밀하게 튜닝해보기

#### 단계
1. **a.** `make_moons(n_samples=1000, noise=0.4)`로 데이터셋 생성
2. **b.** `train_test_split()`으로 훈련/테스트 세트로 분할 (`test_size=0.2`)
3. **c.** `GridSearchCV`로 다양한 하이퍼파라미터를 교차 검증과 함께 그리드 탐색
   - 탐색한 파라미터:
     - `max_leaf_nodes`: 2 ~ 99
     - `max_depth`: 1 ~ 6
     - `min_samples_split`: 2, 3, 4
   - `cv=3` (3-fold CV)
4. **d.** 찾은 최적 파라미터로 전체 훈련 세트에 학습 후 테스트 세트로 평가
   - **최적 파라미터**: `{'max_depth': 2, 'max_leaf_nodes': 4, 'min_samples_split': 2}`
   - **테스트 정확도**: **0.855** (약 85~87% 범위에 부합)

#### (추가) Best Model 시각화
- **결정 경계 시각화**: `meshgrid`로 격자를 만들고 `predict`해서 등고선으로 시각화
  - moons 데이터의 두 클래스를 가르는 계단 모양 결정 경계 확인 가능
- **학습된 트리 구조 시각화**: `export_graphviz` + `Source`로 트리를 그래프로 표현
  - 루트: `x2 ≤ 0.35`로 분할
  - 깊이 1: 왼쪽은 `x1 ≤ -0.366`, 오른쪽은 `x1 ≤ 1.548`
  - 4개의 리프 노드로 결과 분류
  - `max_leaf_nodes=4`, `max_depth=2` 라는 최적 파라미터와 정확히 일치하는 단순한 트리
