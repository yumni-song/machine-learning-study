## 10.1 생물학적 뉴런에서 인공 뉴런까지

### 인공 신경망이 부상하게 된 배경

인공 신경망(ANN)은 우리 생활에 훨씬 커다란 영향을 미칠 것으로 예상되며, 그 이유는 다음과 같다.

- **데이터 규모의 폭발적 증가**: 신경망을 훈련하기 위한 데이터가 엄청나게 많아짐.
- **하드웨어의 발전**: 1990년대 이후 컴퓨터 하드웨어(특히 GPU)가 크게 발전.
- **훈련 알고리즘의 향상**: 역전파, 최적화 기법 등 학습 알고리즘이 개선됨.
- **이론적 제약 극복**: 일부 인공 신경망의 이론상 제한이 실전에서는 문제가 되지 않는다고 밝혀짐.
- **투자–진보의 선순환**: 성과 → 투자 → 더 큰 성과로 이어지는 사이클에 진입.

### 10.1.1 생물학적 뉴런

인공 신경망의 모티브가 된 **생물학적 신경망(Biological Neural Network, BNN)** 의 구조에 대한 연구가 활발히 진행되어 왔다.

**그림 10-1 (생물학적 뉴런)**: 하나의 뉴런은 다음과 같은 구성요소로 이루어진다.

- **세포체(soma)**: 핵, 미토콘드리아, 소포체, 골지체 등을 포함하는 뉴런의 본체.
- **수상돌기(dendrite)**: 다른 뉴런으로부터 신호를 받는 입력 통로. 여러 갈래의 가지가 존재.
- **축삭돌기(axon)**: 세포체에서 길게 뻗어 나와 다른 뉴런으로 신호를 전달하는 통로.
- **축삭둔덕(axon hillock)**: 세포체와 축삭이 연결되는 부분으로, 활동전위가 발생하는 위치.
- **축삭끝가지 / 시냅스 말단(synaptic terminal)**: 다른 뉴런의 수상돌기와 연결되어 신호를 전달하는 부위.

**그림 10-2 (사람의 피질)**: 수많은 뉴런들이 여러 층(layer)으로 복잡하게 연결되어 신경망을 형성한다. 이러한 **계층 구조**가 인공 신경망의 다층(layered) 구조의 영감이 되었다.

### 10.1.2 뉴런을 사용한 논리 연산

**인공 뉴런(artificial neuron)** 은 매컬러(McCulloch)와 피츠(Pitts)가 생물학적 뉴런에서 착안해 제안한 매우 단순한 신경망 모델이다. 입력이 활성화 상태이면 출력도 활성화될 수 있고, 이 단순한 모델만으로도 기본적인 논리 연산을 수행할 수 있다.

**그림 10-3 (간단한 논리 연산을 수행하는 인공 신경망)** 의 해석:

| 회로 구조 | 의미하는 연산 | 설명 |
|---|---|---|
| A → C | `C = A` | 항등 연산 (입력이 그대로 출력) |
| A, B → C | `C = A ∧ B` | 논리곱(AND): A, B가 모두 활성화되어야 C 활성화 |
| A, B → C (여러 연결) | `C = A ∨ B` | 논리합(OR): A 또는 B 중 하나만 활성화돼도 C 활성화 |
| A → C, B → C (B는 억제 연결) | `C = A ∧ ¬B` | A는 활성, B는 비활성일 때 C 활성화 |

기호 정의: `∧` = 논리곱(AND), `∨` = 논리합(OR), `¬` = 논리부정(NOT)

이로써 단순한 뉴런 결합만으로 모든 논리 회로(즉, 임의의 계산)를 표현할 수 있음을 보여주었다.

### 10.1.3 퍼셉트론(Perceptron)

**퍼셉트론**은 가장 간단한 인공 신경망 구조로, 1957년에 **프랑크 로젠블라트(Frank Rosenblatt)** 가 제안하였다. 퍼셉트론의 기본 단위는 **TLU(Threshold Logic Unit, 임곗값 논리 유닛)** 이다.

#### TLU의 동작 원리 (그림 10-4)

1. **선형 함수**: 입력 \(x_1, x_2, x_3\) 와 그에 대응하는 가중치 \(w_1, w_2, w_3\) 의 가중합을 구한 뒤 편향 \(b\) 를 더한다.
   $$z = w^T x + b$$
2. **계단 함수(step function)**: 계산된 \(z\) 에 계단 함수를 적용해 출력을 얻는다.
   $$h_{w,b}(x) = \text{step}(w^T x + b)$$

#### 식 10-1: 일반적으로 사용되는 계단 함수 (임곗값 0 가정)

**헤비사이드(Heaviside) 계단 함수**

$$
\text{heaviside}(z) =
\begin{cases}
0, & z < 0 \\
1, & z \geq 0
\end{cases}
$$

**부호 함수(sgn)**

$$
\text{sgn}(z) =
\begin{cases}
-1, & z < 0 \\
0,  & z = 0 \\
+1, & z > 0
\end{cases}
$$

두 함수 모두 입력의 부호에 따라 이산적인 값(0/1 또는 -1/0/+1)을 출력한다는 공통점이 있다. 퍼셉트론에서는 주로 헤비사이드 함수를 사용한다.

---

#### 퍼셉트론의 층 구조

퍼셉트론은 하나의 층 안에 놓인 하나 이상의 TLU로 구성되며, **각각의 TLU는 모든 입력에 연결**된다.

- **완전 연결 층(fully connected layer)** 또는 **밀집 층(dense layer)**: 한 층의 모든 뉴런이 이전 층의 모든 뉴런에 연결된 구조.
- **입력 층(input layer)**: 입력 신호 자체가 모여 있는 층.
- **출력 층(output layer)**: TLU들이 최종 출력을 생성하는 층.

**그림 10-5** 는 두 개의 입력 \(x_1, x_2\) 와 세 개의 출력 뉴런으로 구성된 퍼셉트론을 보여준다. 각 입력은 세 개의 출력 뉴런 모두와 연결되어 있다(완전 연결).

#### 식 10-2: 완전 연결 층의 출력 계산

$$h_{W,b}(X) = \phi(XW + b)$$

- \(X\): 입력 행렬 (행: 샘플, 열: 특성).
- \(W\): 가중치 행렬 (행: 입력 특성, 열: 뉴런).
- \(b\): 편향 벡터 (뉴런마다 하나의 편향).
- \(\phi\): 활성화 함수 (TLU의 경우 계단 함수).
- 결과 \(h_{W,b}(X)\): 출력 행렬.

행렬 곱셈을 사용해 한 번에 여러 샘플과 여러 뉴런의 출력을 계산할 수 있다.

---

#### 헤브의 규칙(Hebb's Rule)과 학습

**헤브의 학습 규칙**은 신경과학의 고전적 원리로 다음 한 문장으로 요약된다.

> "서로 활성화되는 세포가 서로 연결된다."
> (Cells that fire together, wire together.)

즉, **두 뉴런이 동시에 활성화될 때마다 이들 사이의 연결 가중치가 증가하는 경향**이 있다는 의미다. 이 아이디어가 퍼셉트론의 가중치 갱신 규칙에 반영된다.

#### 식 10-3: 퍼셉트론 학습 규칙 (가중치 업데이트)

$$w_{i,j}^{(\text{next step})} = w_{i,j} + \eta \, (y_j - \hat{y}_j) \, x_i$$

- \(w_{i,j}\): \(i\) 번째 입력과 \(j\) 번째 출력 뉴런 사이의 가중치.
- \(x_i\): \(i\) 번째 입력값.
- \(y_j\): \(j\) 번째 뉴런의 **실제 정답**.
- \(\hat{y}_j\): \(j\) 번째 뉴런의 **현재 예측값**.
- \(\eta\): 학습률(learning rate).

**해석**:
- 예측이 정답과 같으면 \((y_j - \hat{y}_j) = 0\) 이므로 가중치는 변하지 않는다.
- 정답이 1인데 0으로 예측했다면 가중치를 증가시켜 다음에는 더 잘 활성화되도록 한다.
- 정답이 0인데 1로 예측했다면 가중치를 감소시켜 활성화를 억제한다.

#### 퍼셉트론 수렴 이론

- **훈련 샘플이 선형적으로 구분될 수 있다면, 퍼셉트론 학습 알고리즘은 반드시 정답에 수렴**한다는 이론.
- 단, 데이터가 선형 분리 가능하지 않으면(예: XOR) 수렴이 보장되지 않는다.

---

#### 다층 퍼셉트론(MLP)으로 한계를 극복하기

단층 퍼셉트론은 **XOR 문제**와 같이 선형 분리가 불가능한 문제는 풀지 못한다. 그러나 **퍼셉트론을 여러 개 쌓아올린 다층 퍼셉트론(MLP)** 을 사용하면 일부 제약을 줄일 수 있다.

**그림 10-6 (XOR 분류 문제와 이를 푸는 다층 퍼셉트론)** 해석:

- 좌측: 2차원 평면에 (0,0), (1,1)은 한 클래스, (1,0), (0,1)은 다른 클래스로 표시되어 있다. 하나의 직선으로 두 클래스를 분리할 수 없음을 보여준다.
- 우측: 두 개의 입력 \(x_1, x_2\) 가 가중치 1로 두 개의 은닉 TLU에 연결되고, 각 은닉 TLU의 편향이 \(-3/2\) 와 \(-1/2\) 로 설정되어 있다. 은닉층의 출력은 가중치 \(-1, 1\) 등을 거쳐 편향 \(-1/2\) 를 가진 출력 TLU로 전달된다.
- 이 구조는 한 은닉 뉴런이 **AND** 역할, 다른 은닉 뉴런이 **OR** 역할을 수행하여 최종적으로 **XOR**을 구현한다.

### 10.1.4 다층 퍼셉트론과 역전파

**다층 퍼셉트론(MLP)** 의 구성:

- **입력 층(input layer)**: 입력 데이터가 들어오는 층.
- **은닉 층(hidden layer)**: 입력 층과 출력 층 사이에 위치한 하나 이상의 TLU 층.
- **출력 층(output layer)**: 최종 출력을 생성하는 층.
- **하위 층(lower layer)**: 입력 층과 가까운 층.
- **상위 층(upper layer)**: 출력에 가까운 층.

**그림 10-7**: 입력 2개, 은닉 뉴런 4개, 출력 뉴런 3개로 구성된 MLP의 구조를 보여준다. 입력층의 모든 뉴런은 은닉층의 모든 뉴런과 연결되고, 은닉층의 모든 뉴런은 출력층의 모든 뉴런과 완전 연결된다.

#### 심층 신경망(DNN)

은닉 층이 여러 개 깊게 쌓인 신경망을 **심층 신경망(deep neural network, DNN)** 이라고 한다.

#### 후진 모드 자동 미분 (reverse-mode autodiff)

심층 신경망의 가중치 학습에는 모든 가중치에 대한 손실 함수의 그레이디언트가 필요하다. **후진 모드 자동 미분**은 이 모든 그레이디언트를 자동·효율적으로 계산하는 알고리즘이다.

#### 역전파(Backpropagation)

**역전파**는 **후진 모드 자동 미분과 경사 하강법을 결합**한 학습 알고리즘이며, 다음 단계로 진행된다.

1. 하나의 **미니배치**(예: 32개 샘플)씩 진행하여 전체 훈련 세트를 한 번 처리한다.
2. 이러한 처리를 여러 번 반복하며, 한 번의 전체 훈련 세트 처리를 **에포크(epoch)** 라고 한다.
3. **정방향 계산(forward pass)**: 입력으로부터 각 층을 순서대로 통과시켜 예측값을 얻는다.
4. 네트워크의 **출력 오차를 측정**하고, **연쇄법칙(chain rule)** 을 사용해 각 가중치가 오차에 기여한 정도(그레이디언트)를 거꾸로 계산한다.
5. **경사 하강법**을 수행하여 방금 계산한 오차 그레이디언트를 사용해 네트워크의 모든 연결 가중치를 수정한다.

---

#### 역전파에 널리 쓰이는 활성화 함수

계단 함수는 미분이 0 또는 정의되지 않아 경사 하강법을 적용할 수 없다. 따라서 미분 가능한 활성화 함수를 사용해야 한다.

- **로지스틱(시그모이드) 함수**: \(\sigma(z) = \dfrac{1}{1 + e^{-z}}\) — 출력 범위 (0, 1).
- **하이퍼볼릭 탄젠트(tanh) 함수**:
  $$\tanh(z) = 2\sigma(2z) - 1$$
  — 출력 범위 (-1, 1). 시그모이드보다 출력 평균이 0에 가까워 학습이 빠른 경우가 많다.
- **ReLU(Rectified Linear Unit) 함수**:
  $$\text{ReLU}(z) = \max(0, z)$$
  — 음수 입력은 0, 양수 입력은 그대로 출력. 계산이 매우 빠르고, 깊은 신경망에서 그레이디언트 소실 문제를 완화한다.

**그림 10-8 (활성화 함수와 도함수)** 해석:

- **왼쪽 그래프 (활성화 함수)**:
  - **헤비사이드**(빨간 실선): 0에서 갑작스럽게 1로 점프하는 계단형.
  - **시그모이드**(녹색 점선): 0~1 사이로 부드럽게 변하는 S자 곡선.
  - **Tanh**(파란 실선): -1~1 사이로 변하는 S자 곡선.
  - **ReLU**(분홍 점-쇄선): \(z<0\) 일 땐 0, \(z\geq0\) 일 땐 \(z\) 자체.

- **오른쪽 그래프 (도함수)**:
  - 헤비사이드의 도함수는 거의 모든 곳에서 0이므로 학습에 부적합.
  - 시그모이드의 도함수는 종 모양으로 최댓값은 0.25이며, |z|가 클수록 0에 가까워져 **그레이디언트 소실** 발생.
  - Tanh의 도함수도 종 모양이며 최댓값은 1.
  - ReLU의 도함수는 \(z<0\) 에서 0, \(z>0\) 에서 1 (계단 모양).

### 10.1.5 회귀를 위한 다층 퍼셉트론

회귀 문제에 MLP를 적용할 때, 사이킷런의 `MLPRegressor` 클래스를 사용할 수 있다.

#### `MLPRegressor`의 특징

- **출력 층에서 활성화 함수를 지원하지 않음** → 출력이 실수 그대로 나옴.
- 기본 손실 함수는 **평균 제곱 오차(MSE)**.
- 훈련 세트에 **이상치가 많은 경우 평균 절대 오차(MAE)** 가 더 좋을 수 있다.
- 또는 MSE와 MAE를 조합한 **후버 손실(Huber loss)** 을 사용한다.

#### 표 10-1: 회귀 MLP의 전형적인 구조

| 하이퍼파라미터 | 일반적인 값 |
|---|---|
| 은닉 층 수 | 문제에 따라 다름 (일반적으로 1~5 사이) |
| 은닉 층의 뉴런 수 | 문제에 따라 다름 (일반적으로 10~100 사이) |
| 출력 뉴런 수 | 예측 차원마다 하나 |
| 은닉 층의 활성화 함수 | ReLU |
| 출력 층의 활성화 함수 | 없음. 또는 (출력이 양수일 때) ReLU/소프트플러스, (출력 범위를 제한할 때) 로지스틱/tanh |
| 손실 함수 | MSE 또는 (이상치가 있다면) 후버 |

---

### 10.1.6 분류를 위한 다층 퍼셉트론

다층 퍼셉트론은 다양한 분류 문제에 적용할 수 있다.

- **이진 분류**: 출력 뉴런 1개 + 시그모이드 활성화 함수. 출력값은 양성 클래스의 확률을 의미.
- **다중 레이블 이진 분류**: 예를 들어 "이 이메일이 스팸인지" + "긴급 메일인지" 와 같이 서로 독립적인 여러 이진 레이블을 동시에 다룬다.
  - 시그모이드 활성화 함수를 가진 **두 개의 출력 뉴런**이 필요.
  - 첫 번째 뉴런은 스팸일 확률, 두 번째 뉴런은 긴급 메일일 확률을 출력.
- **다중 클래스 분류(배타적)**: 클래스마다 출력 뉴런 1개 + **소프트맥스(softmax)** 활성화 함수. 출력값들의 합이 1이 되어 확률 분포를 형성.

**그림 10-9 (현대적 다층 퍼셉트론)** 해석:

- 입력층 \(x_1, x_2\) → 은닉층 (ReLU 활성화) → 출력층 (소프트맥스)
- 입력 2개, 은닉 뉴런 4개(ReLU), 출력 뉴런 3개(소프트맥스)로 구성.
- 출력층 위에 별도의 소프트맥스 박스가 있어, 각 출력 뉴런의 점수를 받아 확률 분포로 변환한다.

#### `MLPClassifier`의 특징

- `sklearn.neural_network` 패키지 아래 위치.
- `MLPRegressor`와 거의 동일하지만, MSE가 아닌 **크로스 엔트로피(cross-entropy)** 손실을 최소화한다.

#### 표 10-2: 분류 MLP의 전형적인 구조

| 하이퍼파라미터 | 이진 분류 | 다중 레이블 분류 | 다중 분류 |
|---|---|---|---|
| 은닉 층 수 | 문제에 따라 다름 (일반적으로 1~5 사이) | 〃 | 〃 |
| 출력 뉴런 수 | 1개 | 이진 레이블마다 1개 | 클래스마다 1개 |
| 출력 층 활성화 함수 | 시그모이드 | 시그모이드 | 소프트맥스 |
| 손실 함수 | 크로스 엔트로피 | 크로스 엔트로피 | 크로스 엔트로피 |

## 10.2 케라스(Keras)로 다층 퍼셉트론 구현하기

### 10.2.1 시퀀셜 API로 이미지 분류기 만들기

#### 케라스로 데이터셋 적재하기

**패션 MNIST(Fashion MNIST)** 데이터셋을 사용한다.

- 이미 뒤섞여 **훈련 세트(60,000개) + 테스트 세트(10,000개)** 로 분할되어 있다.
- 추가로 검증 세트를 위해 훈련 세트의 마지막 5,000개를 분리한다 → 훈련 55,000개 + 검증 5,000개 + 테스트 10,000개.

**사이킷런과 다른 점** (케라스로 MNIST/패션 MNIST를 적재할 때):

- 각 이미지가 **784 크기의 1D 배열이 아니라 28×28 크기의 2D 배열**.
- 픽셀 강도가 **실수(0.0~255.0)가 아니라 정수(0~255)** 로 표현됨.

따라서 다음의 전처리가 필요하다.

- 픽셀 강도를 **255.0으로 나누어 0~1 사이로 정규화**한다(부동소수점으로 변환됨).
- 클래스 인덱스(0~9)에 대응하는 클래스 이름 리스트를 별도로 만들어 둔다:
  `["T-shirt/top", "Trouser", "Pullover", "Dress", "Coat", "Sandal", "Shirt", "Sneaker", "Bag", "Ankle boot"]`

예를 들어 훈련 세트의 첫 번째 이미지의 레이블은 9이며, 이는 **"Ankle boot"** 에 해당한다.

**그림 10-10 (패션 MNIST의 샘플)**: 4행 × 10열의 격자에 다양한 의류 이미지(앵클 부츠, 티셔츠, 풀오버, 드레스, 코트, 샌들, 셔츠, 스니커, 가방 등)가 표시되어 데이터셋의 다양성을 보여준다.

---

#### 시퀀셜 API로 모델 만들기

두 개의 은닉 층으로 이루어진 분류용 MLP를 만드는 순서:

1. **랜덤 시드 설정**: 결과의 재현성을 위해 텐서플로의 랜덤 시드를 고정.
2. **`Sequential` 모델 생성**: 층을 순서대로 쌓는 가장 단순한 모델 구조.
3. **Input 층 추가**: 입력 형태(`shape=[28, 28]`)를 지정.
4. **Flatten 층 추가**: 28×28 2D 이미지를 784 길이의 1D 벡터로 펼침.
5. **첫 번째 Dense 은닉 층 추가**: 뉴런 300개, ReLU 활성화.
6. **두 번째 Dense 은닉 층 추가**: 뉴런 100개, ReLU 활성화.
7. **Dense 출력 층 추가**: 클래스마다 하나씩, 뉴런 10개, 소프트맥스 활성화.

층을 하나씩 추가하지 않고 `Sequential` 생성 시 **층의 리스트를 한꺼번에 전달**할 수도 있다. 이 경우 `Input` 층 대신 첫 번째 층에 `input_shape`을 지정한다.

> 💡 **모델 이름 카운터 재설정**: `tf.keras.backend.clear_session()` 과 `tf.random.set_seed(42)` 를 호출하면 `dense`, `dense_1` 같은 자동 이름 카운터가 초기화된다.

---

#### `summary()` 메서드로 모델 구조 확인

| 층 (Layer) | 출력 형태 (Output Shape) | 파라미터 수 (Param #) |
|---|---|---|
| flatten (Flatten) | (None, 784) | 0 |
| dense (Dense) | (None, 300) | 235,500 |
| dense_1 (Dense) | (None, 100) | 30,100 |
| dense_2 (Dense) | (None, 10) | 1,010 |

- **총 파라미터**: 266,610개 (모두 훈련 가능).
- 파라미터 계산:
  - `dense` 층: \(784 \times 300 + 300 = 235{,}500\) (가중치 + 편향).
  - `dense_1` 층: \(300 \times 100 + 100 = 30{,}100\).
  - `dense_2` 층: \(100 \times 10 + 10 = 1{,}010\).
- `None`은 배치 크기를 의미하며, 어떤 크기든 받을 수 있음.

---

#### 모델의 층과 파라미터에 접근하기

- `model.layers`: 모델에 포함된 모든 층의 리스트.
- `model.get_layer('이름')`: 층의 이름으로 특정 층을 선택.
- 각 층의 파라미터에 접근:
  - `layer.get_weights()` → 가중치와 편향을 반환.
  - `layer.set_weights()` → 가중치와 편향을 새로 지정.
- Dense 층의 경우 `weights`(연결 가중치)와 `biases`(편향)가 모두 포함된다.
  - 예: 첫 은닉 층의 가중치 형태는 (784, 300), 편향은 (300,) 이며 편향의 초깃값은 모두 0.

---

#### 모델 컴파일

모델을 만든 뒤 `compile()` 메서드를 호출하여 손실 함수와 옵티마이저를 지정한다.

- **`"sparse_categorical_crossentropy"`**: 레이블이 정수 하나로 이루어져 있고 클래스가 배타적일 때 사용.
- **`"categorical_crossentropy"`**: 샘플마다 클래스별 타깃 확률(원-핫 벡터)이 있을 때 사용.
- **이진/다중 레이블 이진 분류**: 출력 층에 **소프트맥스 대신 시그모이드**를 사용하고 **`"binary_crossentropy"`** 손실을 사용.
- **옵티마이저**: 예시에서는 `"sgd"`(확률적 경사 하강법)를 사용.
- **지표(metrics)**: 분류기이므로 `metrics=["accuracy"]` 로 정확도를 측정하는 것이 유용.

---

#### 모델 훈련과 평가

- 훈련은 `fit()` 메서드로 수행하며, `epochs=30`, `validation_data=(X_valid, y_valid)` 등을 지정.
- 매 에포크마다 훈련 손실/정확도와 검증 손실/정확도가 출력됨.
- 예시 결과:
  - Epoch 1: 훈련 정확도 76.5%, 검증 정확도 83.3%.
  - Epoch 2: 훈련 정확도 83.3%, 검증 정확도 83.8%.
  - Epoch 30: 훈련 정확도 92.0%, 검증 정확도 88.9%.

`fit()` 메서드가 반환하는 **History 객체**에는 다음이 포함된다.

- `history.params`: 훈련 파라미터.
- `history.epoch`: 수행된 에포크 리스트.
- `history.history`: 에포크별 손실과 지표값을 담은 딕셔너리.

**그림 10-11 (학습 곡선)** 해석:

- 에포크가 증가할수록:
  - 훈련 손실(loss)이 빠르게 감소하고, 훈련 정확도(sparse_categorical_accuracy)는 증가.
  - 검증 손실(val_loss)도 감소하지만 어느 시점부터 정체되거나 증가할 수 있음.
  - 검증 정확도(val_sparse_categorical_accuracy)도 증가하다 정체됨.
- 훈련 곡선과 검증 곡선이 멀어지기 시작하면 **과대적합(overfitting)** 의 신호.

---

#### 테스트 세트 평가와 예측

- **`evaluate()` 메서드**: 테스트 세트로 모델을 평가하여 일반화 오차를 추정.
  - 예시: 테스트 손실 0.3243, 테스트 정확도 88.64%.
- **`predict()` 메서드**: 새로운 샘플에 대해 클래스별 확률을 예측 (소프트맥스 출력).
  - 예: 첫 번째 샘플의 확률 분포에서 마지막 클래스가 0.97로 가장 높음 → "Ankle boot"로 분류.
- **`argmax(axis=-1)`**: 각 샘플에 대해 가장 높은 확률을 가진 클래스의 인덱스를 도출.
  - 예측 결과 `[9, 2, 1]` → `["Ankle boot", "Pullover", "Trouser"]`.

**그림 10-12 (올바르게 분류된 패션 MNIST 이미지)**: 세 개의 테스트 샘플(앵클 부츠, 풀오버, 바지)이 모두 올바르게 분류된 것을 시각적으로 확인할 수 있다.

### 10.2.2 시퀀셜 API로 회귀용 다층 퍼셉트론 만들기

**캘리포니아 주택 가격 데이터셋**으로 회귀 문제를 푸는 MLP를 만든다.

#### 모델 구조

- **정규화 층(`Normalization`)**: 입력 특성의 평균을 0, 표준편차를 1로 정규화. 학습 전에 `norm_layer.adapt(X_train)` 으로 훈련 데이터의 통계량을 학습시켜야 한다.
- **3개의 은닉 층**: 각각 50개의 뉴런, ReLU 활성화.
- **출력 층**: 뉴런 1개, 활성화 함수 없음(회귀이므로 실수 출력).

#### 컴파일과 학습 설정

- **옵티마이저**: Adam (학습률 1e-3).
- **손실 함수**: `"mse"` (평균 제곱 오차).
- **평가 지표**: `"RootMeanSquaredError"` (RMSE).
- **에포크**: 20.

#### 결과 예시

- 테스트 RMSE: 약 0.535.
- 새로운 샘플에 대한 예측: [0.527, 1.001, 5.150] 등의 실수 값을 출력.

## 10.3 신경망 하이퍼파라미터 튜닝하기

### 하이퍼파라미터 튜닝의 방법

- **사이킷런의 `GridSearchCV` / `RandomizedSearchCV`** 를 사용하여 미세 튜닝할 수 있다.
- **케라스 튜너(Keras Tuner)** 라이브러리를 사용하면 더 효율적인 탐색이 가능 (별도 설치 필요).

#### 패션 MNIST용 MLP 모델 생성 함수 (`build_model(hp)`)

튜닝할 하이퍼파라미터:

- **`n_hidden`**: 은닉 층의 수 (정수, 최소 0, 최대 8, 기본값 2).
- **`n_neurons`**: 각 층의 뉴런 수 (정수, 최소 16, 최대 256).
- **`learning_rate`**: 학습률 (실수, 1e-4 ~ 1e-2, 로그 샘플링).
- **`optimizer`**: 옵티마이저 (`"sgd"` 또는 `"adam"` 중 선택).

선택된 옵티마이저에 따라 SGD 또는 Adam이 생성되고, 모델은 입력 Flatten → 여러 개의 Dense(ReLU) → Dense(10, softmax) 구조로 구성된다.

---

### 랜덤 서치 수행

- **`kt.RandomSearch`** 튜너를 사용한다.
- 주요 인자:
  - `objective="val_accuracy"`: 검증 정확도를 최대화 목표.
  - `max_trials=5` (또는 10): 시도 횟수.
  - `overwrite=True`: 이전 결과 덮어쓰기.
  - `directory`, `project_name`: 결과 저장 경로.
  - `seed`: 재현성을 위한 시드.

`search()` 메서드를 호출하면 지정된 횟수만큼 다양한 하이퍼파라미터 조합으로 모델을 훈련하고 검증한다.

### 최상의 결과 추출

- **`get_best_models(num_models=3)`**: 가장 성능이 좋은 상위 N개의 모델을 반환.
- **`get_best_hyperparameters(num_trials=3)`**: 최상의 하이퍼파라미터 값들을 반환.
  - 예: `{'n_hidden': 5, 'n_neurons': 70, 'learning_rate': 0.000412..., 'optimizer': 'adam'}`.

### 오라클(Oracle)

각 튜너는 **오라클의 안내**를 받아 다음 시도할 하이퍼파라미터를 결정한다. 오라클은 모든 시도를 기록하기 때문에:

- `tuner.oracle.get_best_trials(num_trials=1)[0]`: 최상의 시도를 가져옴.
- `best_trial.summary()`: 해당 시도의 하이퍼파라미터와 점수를 요약 출력.
- `best_trial.metrics.get_last_value("val_accuracy")`: 검증 정확도 등의 최종 지표값에 접근.

### 모델 최종화 및 배포

검증 성능이 만족스러우면:

1. 전체 훈련 세트(`X_train_full`, `y_train_full`)에서 추가로 몇 번의 에포크 동안 이어서 훈련.
2. 테스트 세트에서 최종 평가(`evaluate()`).
3. 제품 환경에 배포.

### 10.3.1 은닉 층 개수

**핵심 개념**: 복잡한 문제에서는 **심층 신경망**이 얕은 신경망보다 **파라미터 효율성(parameter efficiency)** 이 훨씬 좋다.

#### 이유: 실제 데이터의 계층 구조

실제 데이터는 계층 구조를 가진 경우가 많아 심층 신경망이 유리하다.

- **아래쪽 은닉 층**: **저수준의 구조**를 모델링 (예: 여러 방향이나 모양의 선).
- **중간 은닉 층**: 저수준의 구조를 연결해 **중간 수준의 구조**를 모델링 (예: 사각형, 원).
- **가장 위쪽 은닉 층과 출력 층**: 중간 수준의 구조를 연결해 **고수준의 구조**를 모델링 (예: 얼굴).

#### 장점

- 계층 구조는 심층 신경망이 **좋은 솔루션으로 빠르게 수렴**하도록 도와준다.
- **새로운 데이터에 일반화되는 능력**도 향상된다.
- **전이 학습(transfer learning)**: 한 작업에서 학습된 저수준/중간수준 표현을 다른 유사한 작업에 재사용할 수 있다.

---

### 10.3.2 은닉 층의 뉴런 개수

#### 입력층/출력층 뉴런 수

- 입력과 출력의 **형태**에 따라 결정된다.
- 예: MNIST는 28×28 = **784개의 입력 뉴런**과 (10개 클래스이므로) **10개의 출력 뉴런**이 필요.

#### 은닉층 뉴런 수의 전통적 구성

- 일반적으로 각 층의 뉴런을 점점 줄여서 **깔때기(funnel)** 처럼 구성한다.
- 이유: 저수준의 많은 특성이 고수준의 적은 특성으로 합쳐질 수 있기 때문.

#### 현실적인 권장 접근

- 층의 개수와 마찬가지로, **과대적합이 시작되기 전까지 점진적으로 뉴런 수를 늘리는 방식**을 사용한다.
- 더 실용적인 방법: **필요한 것보다 더 많은 층과 뉴런을 가진 모델을 선택**한 다음, **조기 종료(early stopping)나 규제(regularization)** 기법으로 과대적합을 방지한다.

### 10.3.3 학습률, 배치 크기 그리고 다른 하이퍼파라미터

| 하이퍼파라미터 | 권장 사항 / 설명 |
|---|---|
| **학습률(learning rate)** | 일반적으로 최적의 학습률은 **최대 학습률**(훈련 알고리즘이 발산하는 학습률)의 **절반 정도**. |
| **옵티마이저(optimizer)** | 고전적인 평범한 미니배치 경사 하강법보다 **더 좋은 옵티마이저(Adam 등)** 를 선택하는 것이 매우 중요하며, 그 옵티마이저의 하이퍼파라미터도 튜닝 필요. |
| **배치 크기(batch size)** | 모델 성능과 훈련 시간에 큰 영향. **GPU RAM에 맞는 가장 큰 배치 크기** 사용이 권장됨. |
| **활성화 함수(activation function)** | 일반적으로 **ReLU**가 모든 은닉 층에 좋은 기본값. **출력 층의 활성화 함수는 수행하는 작업에 따라 달라짐** (회귀: 없음/ReLU/tanh, 이진 분류: 시그모이드, 다중 분류: 소프트맥스). |
| **반복 횟수(epochs)** | 대부분의 경우 튜닝할 필요 없음. 대신 **조기 종료(early stopping)** 를 사용. |

## 연습문제 10. 심층 다층 퍼셉트론을 MNIST 데이터셋에 적용하기

### 문제

> 심층 다층 퍼셉트론을 MNIST 데이터셋에 훈련해 보세요. `tf.keras.datasets.mnist.load_data()` 함수를 사용해 데이터를 적재할 수 있습니다. **98% 이상의 정확도**를 얻을 수 있는지 확인해 보세요. 이 장에서 소개한 방법(**keras_tuner**)을 사용해 최적의 파라미터를 찾아 보세요.

### 풀이의 흐름

1. **데이터셋 로드 및 전처리**
   - MNIST 데이터(60,000개 훈련 + 10,000개 테스트)를 로드.
   - 픽셀값을 255.0으로 나누어 정규화.
   - 훈련 세트의 마지막 5,000개를 분리해 검증 세트로 사용.

2. **모델 생성 함수 `build_model(hp)` 정의**
   - 튜닝 대상 하이퍼파라미터:
     - `n_hidden` (은닉 층 수): 0~8, 기본 2.
     - `n_neurons` (뉴런 수): 16~256.
     - `learning_rate`: 1e-4 ~ 1e-2 (로그 샘플링).
     - `optimizer`: `"sgd"` 또는 `"adam"`.
   - 입력: Flatten(28×28 → 784)
   - 은닉층: `n_hidden` 개의 Dense 층 (각 `n_neurons` 개, ReLU)
   - 출력층: Dense(10, softmax)
   - 손실 함수: `sparse_categorical_crossentropy`, 지표: 정확도.

3. **랜덤 서치 수행 (`kt.RandomSearch`)**
   - `max_trials=10`, `objective="val_accuracy"`.
   - 각 시도마다 10 에포크 훈련 후 검증 정확도로 평가.

4. **최상의 모델/하이퍼파라미터 획득**
   - `get_best_models(num_models=3)` 으로 상위 3개의 모델 추출.
   - `get_best_hyperparameters(num_trials=3)` 으로 최상의 하이퍼파라미터를 얻음.
   - **결과 예시**:
     - `n_hidden`: **6**
     - `n_neurons`: **187**
     - `learning_rate`: **약 0.00179**
     - `optimizer`: **adam**
     - 검증 정확도: **약 97.82%**

5. **오라클을 통한 시도 요약**
   - `best_trial.summary()` 로 최상의 시도의 하이퍼파라미터와 점수를 출력.
   - `best_trial.metrics.get_last_value("val_accuracy")` 로 최종 검증 정확도를 확인.

6. **전체 훈련 세트로 최종 학습 및 테스트 평가**
   - 전체 훈련 세트(`X_train_full`)에서 30 에포크 동안 이어서 훈련.
   - 테스트 세트에서 평가 → **테스트 정확도 약 97.79%** 달성.

### 결과 해석

- 목표였던 **98% 이상의 정확도**에는 약간 못 미쳤지만(97.79%), 매우 근접한 결과를 얻었다.
- 추가로 정확도를 향상시키려면:
  - 시도 횟수(`max_trials`)와 에포크 수를 늘리기.
  - 배치 정규화, 드롭아웃 등 규제 기법 추가.
  - 합성곱 신경망(CNN) 사용 등 모델 구조 자체를 변경.

## 정리: 신경망 학습의 전체 흐름

1. **모델 정의**: 입력 → 은닉 층(ReLU) → 출력 층(작업에 맞는 활성화).
2. **컴파일**: 손실 함수(MSE/크로스엔트로피) + 옵티마이저(Adam 등) + 지표(정확도/RMSE).
3. **훈련**: `fit()` 호출, 미니배치 + 에포크, 검증 세트로 과대적합 모니터링.
4. **평가**: `evaluate()` 로 테스트 세트에서 일반화 오차 측정.
5. **예측**: `predict()` 로 새로운 샘플에 대한 출력 생성.
6. **튜닝**: Keras Tuner의 RandomSearch 등을 활용해 하이퍼파라미터 최적화.
7. **배포**: 전체 훈련 데이터로 최종 학습 후 제품 환경에 배포.
