## 3.1 MNIST 데이터셋
* **개요**: 70,000개의 숫자 손글씨 이미지 데이터셋 (학습 60,000 / 테스트 10,000).
* **데이터 구조 (사이킷런)**:
    - `DESCR`: 데이터셋 설명.
    - `data`: 2D 넘파이 배열 (784개 특성: $28 \times 28$ 픽셀).
    - `target`: 레이블 (0~9).
* **시각화**: 샘플의 특성 벡터를 $28 \times 28$ 배열로 `reshape`한 뒤 `imshow()`로 출력.

## 3.3 성능 측정 (Performance Measures)

### 3.3.1 교차 검증 및 정확도
* **정확도의 함정**: 불균형한 데이터셋(예: 5는 전체의 10%)에서는 모든 것을 '5 아님'으로 예측해도 90%의 정확도가 나오므로 신뢰하기 어려움.

### 3.3.2 오차 행렬 (Confusion Matrix)
* **개념**: 클래스 A가 클래스 B로 분류된 횟수를 표로 나타냄.
* **지표**:
    - **정밀도 (Precision)**: 양성으로 예측한 것 중 실제 양성의 비율.
      $$Precision = \frac{TP}{TP + FP}$$
    - **재현율 (Recall)**: 실제 양성 중 정확히 감지한 비율 (민감도).
      $$Recall = \frac{TP}{TP + FN}$$
    - **F1 점수**: 정밀도와 재현율의 조화 평균.
      $$F1 = \frac{TP}{TP + \frac{FN + FP}{2}}$$



### 3.3.3 정밀도/재현율 트레이드오프
* **결정 임계값(Threshold)**:
    - 임계값을 **높이면**: 정밀도 상승, 재현율 하락.
    - 임계값을 **낮추면**: 정밀도 하락, 재현율 상승.
* **PR 곡선**: 재현율에 대한 정밀도 곡선으로 모델의 성능을 비교.

### 3.3.4 ROC 곡선과 AUC
* **ROC 곡선**: FPR(거짓 양성 비율)에 대한 TPR(재현율)의 곡선.
* **AUC (Area Under Curve)**: 곡선 아래 면적. 1에 가까울수록 완벽한 모델이며, 0.5는 완전한 랜덤 분류기.

## 3.4 다중 분류 (Multiclass Classification)
* **전략**:
    - **OvR (One-versus-the-Rest)**: 숫자 하나마다 분류기를 만들어(총 10개) 점수가 가장 높은 클래스 선택.
    - **OvO (One-versus-One)**: 모든 숫자 조합(0-1, 0-2 등)에 대해 분류기 훈련 ($N \times (N-1) / 2$개).
* **사이킷런**: 이진 분류 알고리즘을 사용하면 자동으로 OvR 또는 OvO를 실행.

---

## 3.5 오류 분석 (Error Analysis)
* **시각화**: 오차 행렬을 컬러 그래프로 나타내어 어디서 오류가 발생하는지 확인.
* **정규화**: 행/열 단위로 정규화하여 절대 개수가 아닌 **에러 비율**을 분석.
* **개선**: 에러가 잦은 클래스(예: 3과 5)에 대해 **데이터 증식(Data Augmentation)** 등을 통해 훈련 데이터 보강.

---

## 3.6 다중 레이블 분류 (Multilabel Classification)
* **개념**: 한 샘플에 여러 개의 태그를 할당 (예: [7보다 큰가?, 홀수인가?]).
* **평가**: 각 레이블의 F1 점수를 평균 내거나 가중 평균을 사용.
* **체인(ClassifierChain)**: 레이블 간의 의존성을 고려하기 위해 이전 모델의 예측값을 다음 모델의 입력으로 사용.

---

## 3.7 다중 출력 분류 (Multioutput Classification)
* **개념**: 다중 레이블에서 한 레이블이 다중 클래스 값을 가질 수 있는 경우.
* **사례**: 이미지 잡음 제거 시스템.
    - 입력: 잡음 섞인 MNIST 이미지.
    - 출력: 깨끗한 원본 이미지 (각 픽셀(784개 레이블)이 0~255 값을 가짐).
