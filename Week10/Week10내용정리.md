## 7.1 투표 기반 분류기 (Voting Classifier)

### 앙상블 학습 (Ensemble Learning)

- **앙상블 학습**이란 일련의 예측기(분류 또는 회귀 모델)로부터 예측을 수집하여, 가장 좋은 모델 하나보다 더 나은 예측을 얻는 방법이다.
- 이를 **"wisdom of crowd(대중의 지혜)"** 라고 부른다.
- 대표적 예시: **랜덤 포레스트는 결정 트리의 앙상블**이다.

### 투표 기반 분류기 (Voting Classifier)

- 정확도가 80% 정도인 분류기 여러 개 (로지스틱 회귀, SVM, 랜덤 포레스트, K-최근접 이웃 등)를 훈련시켰다고 가정하자.
- **더 좋은 분류기를 만드는 가장 간단한 방법은 각 분류기의 예측을 집계(aggregate)하는 것**이다.

### 직접 투표 (Hard Voting)

- **가장 많은 표를 얻은 클래스를 앙상블의 예측으로 선택**하는 방식.
- 다수결 투표 분류기는 앙상블에 포함된 개별 분류기 중 가장 뛰어난 것보다도 정확도가 높은 경우가 많다.

#### 약한 학습기와 강한 학습기

- **약한 학습기 (weak learner)**: 랜덤 추측보다 조금 더 높은 성능을 내는 분류기
- **강한 학습기 (strong learner)**: 높은 정확도를 내는 분류기
- 약한 학습기가 충분히 많고 다양하다면, 앙상블은 강한 학습기가 될 수 있다.

### 큰 수의 법칙 (Law of Large Numbers)

- 균형이 틀어진 동전(앞면 51%, 뒷면 49%)을 던질 때, **던진 횟수가 증가할수록 앞면이 나올 비율은 51%에 가까워진다**.
- 즉, 시행을 충분히 많이 하면 통계적 기댓값에 수렴한다.

#### 앙상블에 적용하면

- 랜덤 추측보다 조금 더 나은 다수의 분류기로 앙상블을 구축하면 전체 예측은 진짜 클래스에 수렴할 가능성이 높아진다.
- **단, 예측기들이 서로 독립적일 때(상관관계가 없을 때) 최고의 성능을 발휘**한다.
- 다양한 분류기를 얻기 위한 한 가지 방법은 **각기 다른 알고리즘으로 학습**시키는 것이다.

#### 그림 7-3 해석 (큰 수의 법칙 그래프)

- x축: 동전을 던진 횟수
- y축: 앞면이 나온 비율
- 시행 횟수가 적을 때(왼쪽)는 0.42 ~ 0.58 사이에서 매우 변동성이 크지만, 시행 횟수가 많아질수록(오른쪽) 모든 곡선이 51% 근처로 수렴함을 보여준다.
- 이는 약한 학습기를 많이 모으면 앙상블이 안정적인 정확도를 가질 수 있다는 직관을 제공한다.

### 간접 투표 (Soft Voting)

- 모든 분류기가 클래스의 확률을 예측할 수 있으면(즉, `predict_proba()` 메서드가 있으면), **개별 분류기의 예측 확률을 평균 내어 가장 높은 확률의 클래스를 예측**한다.
- **확률이 높은 투표에 더 비중을 두므로 직접 투표 방식보다 일반적으로 성능이 높다**.
- `voting` 매개변수를 `"soft"`로 바꾸고, 모든 분류기가 클래스 확률을 추정할 수 있어야 한다.
- SVC는 기본값에서 클래스 확률을 제공하지 않으므로 `probability=True`로 지정해야 한다.

## 7.2 배깅과 페이스팅 (Bagging and Pasting)

### 개념

다양한 분류기를 만드는 또 다른 방법은 **같은 알고리즘을 사용하면서 훈련 세트의 서브셋을 랜덤으로 구성하여 분류기를 다르게 학습**시키는 것이다.

### 배깅 vs. 페이스팅

| 구분 | 설명 |
| --- | --- |
| **배깅 (Bagging)** | bootstrap aggregating의 줄임말. 훈련 세트에서 **중복을 허용하여** 샘플링 |
| **페이스팅 (Pasting)** | **중복을 허용하지 않고** 샘플링 |

### 예측 집계 방식

- **분류**: 통계적 최빈값 (statistical mode) — 가장 많이 등장한 클래스
- **회귀**: 평균 (mean)

### 배깅·페이스팅의 효과

- 일반적으로 앙상블 결과는 원본 데이터셋으로 단일 예측기를 훈련시킬 때와 비교하여 **편향(bias)은 비슷하지만 분산(variance)은 줄어든다**.
- **높은 확장성**: 학습과 예측을 병렬로 수행할 수 있다.

#### 그림 7-4 해석

- 하나의 훈련 세트에서 여러 개의 서로 다른 부분집합(subset)을 만들고, 각 부분집합으로 별도의 예측기를 훈련시키는 구조를 보여준다.
- 모든 예측기가 동일한 알고리즘이지만 훈련 데이터가 달라 서로 다른 결과를 내며, 이를 집계해서 최종 예측을 만든다.

### 7.2.1 사이킷런의 배깅과 페이스팅

- 클래스: `BaggingClassifier` (회귀는 `BaggingRegressor`)
- 주요 매개변수:
  - `n_estimators`: 예측기 개수
  - `max_samples`: 각 예측기가 사용할 샘플 수
  - `bootstrap=True` → 배깅, `bootstrap=False` → 페이스팅
  - `n_jobs`: 사이킷런이 훈련/예측에 사용할 CPU 코어 수 (`-1`은 모든 코어 사용, 기본값 1)

### 단일 결정 트리 vs. 배깅 앙상블

#### 그림 7-5 해석 (결정 경계 비교)

- **왼쪽 (단일 결정 트리)**: 결정 경계가 매우 들쭉날쭉하고 복잡하다. 훈련 데이터에 과대적합되는 경향이 있다.
- **오른쪽 (500개 트리의 배깅 앙상블)**: 결정 경계가 훨씬 부드럽고 일반화 성능이 좋다.
- **앙상블의 예측이 결정 트리 하나의 예측보다 일반화가 훨씬 잘 된다**.

### 배깅 vs. 페이스팅 — 어느 쪽이 선호되는가?

- **앙상블에서는 일반적으로 배깅이 페이스팅보다 선호**된다.
- 배깅은 각 예측기가 학습하는 서브셋에 다양성을 추가하므로 페이스팅보다 편향이 조금 더 높다.
- 그러나 **예측기들 간의 상관관계를 줄여 앙상블 전체의 분산을 더 많이 줄인다**.

### 7.2.2 OOB 평가 (Out-of-Bag Evaluation)

- `BaggingClassifier`는 기본값으로 중복을 허용해(`bootstrap=True`) 훈련 세트 크기 m개의 샘플을 선택한다.
- 이 경우 **평균적으로 각 예측기는 훈련 샘플의 약 63%만 학습**하게 된다.
- 선택되지 않은 나머지 **약 37%의 샘플**을 **OOB(out-of-bag) 샘플**이라고 한다.

#### 왜 63%인가?

- m개 샘플에서 m번 중복 추출할 때, 특정 샘플이 한 번도 뽑히지 않을 확률은:
  $$\left(1 - \frac{1}{m}\right)^m \xrightarrow{m \to \infty} \frac{1}{e} \approx 0.368$$
- 따라서 뽑힐 확률은 약 1 − 0.368 = 0.632 ≈ 63%.

### OOB 평가의 장점

- 각 예측기는 자신이 보지 못한 OOB 샘플로 평가가 가능하므로, **별도의 검증 세트를 분리하지 않고도 일반화 성능을 추정**할 수 있다.
- `oob_score=True`로 지정하면 훈련이 끝난 후 자동으로 OOB 평가를 수행하고, 결과는 `oob_score_`에 저장된다.
- `oob_decision_function_`을 통해 각 OOB 샘플에 대한 클래스별 결정 함수의 값(확률)도 확인 가능하다.

## 7.3 랜덤 패치와 랜덤 서브스페이스

`BaggingClassifier`는 샘플뿐 아니라 **특성(feature) 샘플링**도 지원한다.
- 매개변수: `max_features`, `bootstrap_features`
- 훈련 속도를 크게 높일 수 있어 **고차원 데이터셋에 유용**하다.

### 두 가지 방식

| 방식 | 설명 |
| --- | --- |
| **랜덤 패치 방식 (random patches method)** | 훈련 **샘플과 특성을 모두** 샘플링 |
| **랜덤 서브스페이스 방식 (random subspaces method)** | 샘플은 모두 사용(`bootstrap=False`, `max_samples=1.0`)하고, **특성만** 샘플링(`bootstrap_features=True` 또는 `max_features < 1.0`) |

### 효과

- **특성 샘플링은 더 다양한 예측기를 만들어 편향은 늘리지만 분산을 더 낮춘다**.

## 7.4 랜덤 포레스트 (Random Forest)

- 일반적으로 **배깅(또는 페이스팅)을 적용한 결정 트리의 앙상블**이다.
- `BaggingClassifier`에 `DecisionTreeClassifier`를 넣는 대신, 결정 트리에 최적화된 `RandomForestClassifier`를 사용하는 것이 일반적.
- 회귀용: `RandomForestRegressor`

### 랜덤 포레스트의 추가적 무작위성

- 트리의 노드를 분할할 때, **전체 특성 중에서 최선의 특성을 찾는 대신, 랜덤으로 선택한 특성 후보 중에서 최적의 특성을 찾는다**.
- 기본값으로 $\sqrt{n}$개의 특성을 선택 ($n$은 전체 특성 개수).
- 이는 트리들 간의 상관관계를 줄여 앙상블의 분산을 더 낮춘다.

### 7.4.1 엑스트라 트리 (Extra-Trees)

- **익스트림 랜덤 트리(extremely randomized trees) 앙상블**, 줄여서 **엑스트라 트리(extra-trees)**.
- 일반 랜덤 포레스트보다 더 극단적으로 무작위한 트리들로 구성된다.
- **차이점**: 일반 랜덤 포레스트는 후보 특성 중에서 **최적의 임곗값(threshold)을 찾지만**, 엑스트라 트리는 **임곗값을 랜덤으로 분할한 다음 그 중에서 최상의 분할을 선택**한다.
- 이로 인해 편향은 다소 늘어나지만 분산은 줄어들고, **최적 임곗값을 찾는 데 시간을 쓰지 않으므로 일반 랜덤 포레스트보다 훨씬 빠르다**.
- 클래스: `ExtraTreesClassifier`

### 7.4.2 특성 중요도 (Feature Importance)

- 랜덤 포레스트의 또 다른 큰 장점은 **특성의 상대적 중요도를 측정하기 쉽다**는 점이다.
- 훈련이 끝난 뒤, 사이킷런은 각 특성의 중요도를 자동으로 계산하고 **전체 합이 1이 되도록 정규화**한다.
- 이 값은 `feature_importances_` 변수에 저장된다.

#### iris 데이터셋 결과 해석

| 특성 | 중요도 |
| --- | --- |
| sepal length (cm) | 0.11 |
| sepal width (cm) | 0.02 |
| petal length (cm) | 0.44 |
| petal width (cm) | 0.42 |

→ **꽃잎(petal) 정보가 꽃받침(sepal) 정보보다 훨씬 중요**하다는 것을 알 수 있다.

#### 그림 7-6 해석 (MNIST 픽셀 중요도)

- MNIST 데이터에 랜덤 포레스트를 학습시킨 뒤, 각 픽셀(28×28)의 중요도를 시각화한 히트맵.
- **이미지 가운데 부분(특히 숫자가 그려지는 영역)이 가장 밝게(매우 중요함) 나타나고**, 가장자리는 어둡다(중요하지 않음).
- 즉, 모델이 숫자 분류에 중요한 영역을 자동으로 학습했음을 보여준다.

## 7.5 부스팅 (Boosting)

### 개념

- **부스팅**(원래는 가설 부스팅, hypothesis boosting): **약한 학습기를 여러 개 순차적으로 연결하여 강한 학습기를 만드는 앙상블 방법**.
- 핵심: **앞의 모델을 보완해 나가면서** 일련의 예측기를 학습시킨다.
- 가장 인기 있는 방법: **AdaBoost** (adaptive boosting)와 **그레이디언트 부스팅** (Gradient Boosting).

### 7.5.1 AdaBoost

- **이전 모델이 과소적합했던 훈련 샘플의 가중치를 더 높이는 방식**.
- 새로운 예측기는 학습하기 어려운 샘플에 점점 더 집중하게 된다.

#### 학습 과정

1. 첫 번째 분류기를 훈련 세트에 훈련시키고 예측을 만든다.
2. **잘못 분류된 훈련 샘플의 가중치를 상대적으로 높인다**.
3. 두 번째 분류기는 업데이트된 가중치를 사용해 훈련하고 다시 예측을 만든다.
4. 다시 가중치를 업데이트하는 방식으로 반복한다.

#### 그림 7-7 해석

- 동일한 데이터에 대해 첫 번째 예측기 → 두 번째 예측기 → 세 번째 예측기로 이어지면서, 이전 모델이 틀린 샘플(가중치가 높아진 샘플)을 다음 모델이 더 잘 맞추도록 결정 경계가 변화하는 모습을 보여준다.

#### 그림 7-8 해석 (학습률에 따른 변화)

- 두 그래프 모두 moons 데이터셋에 5개의 RBF 커널 SVM 분류기를 순차적으로 훈련한 결과.
- **왼쪽 (`learning_rate = 1`)**: 결정 경계가 빠르게 크게 변화한다.
- **오른쪽 (`learning_rate = 0.5`)**: 학습률을 절반으로 낮추면, 같은 일련의 예측기여도 결정 경계의 변화가 더 완만하다.
- 즉, **학습률이 낮을수록 각 예측기가 이전 모델의 오류를 보정하는 정도가 작아지므로 더 많은 예측기가 필요하지만, 일반화 성능은 좋아질 수 있다**.

### AdaBoost 핵심 수식 정리 (자세히)

AdaBoost의 동작 원리를 이해하려면 **세 가지 수식**과 그 의미를 정확히 파악해야 한다.

#### 식 7-1: j번째 예측기의 가중치가 적용된 오류율

$$
r_j = \frac{\displaystyle\sum_{\substack{i=1 \\ \hat{y}_j^{(i)} \neq y^{(i)}}}^{m} w^{(i)}}{\displaystyle\sum_{i=1}^{m} w^{(i)}}
$$

**의미**:
- 분자: **잘못 분류된 샘플들의 가중치 합** (예측 $\hat{y}_j^{(i)}$ 가 실제 $y^{(i)}$ 와 다른 경우만 더함)
- 분모: 모든 샘플의 가중치 합 (정규화)
- 즉, $r_j$는 **"가중치를 고려한 오류율"** 이다.
- 모든 샘플이 동일한 가중치를 가진다면 일반적인 오류율(틀린 비율)과 같지만, AdaBoost는 어려운 샘플의 가중치가 점점 커지므로 일반 오류율과는 다른 의미를 갖는다.

---

#### 식 7-2: 예측기 가중치 (predictor weight)

$$
\alpha_j = \eta \log \frac{1 - r_j}{r_j}
$$

여기서 $\eta$는 **학습률 하이퍼파라미터**.

**$\alpha_j$의 의미와 동작 (★ 자세히)**:

| 오류율 $r_j$ 의 상태 | $\frac{1-r_j}{r_j}$ | $\alpha_j = \eta \log \frac{1-r_j}{r_j}$ | 해석 |
| --- | --- | --- | --- |
| $r_j$ 가 매우 작음 (예측기가 정확함) | 매우 큼 ($\gg 1$) | **큰 양수** | **이 예측기에 큰 영향력 부여** |
| $r_j \approx 0.5$ (랜덤 추측 수준) | $\approx 1$ | $\log 1 = 0$ → **0에 가까움** | 거의 영향력 없음 |
| $r_j > 0.5$ (랜덤 추측보다 못함) | $< 1$ | **음수** | 음의 영향력 (반대 방향 사용) |

→ **예측기가 정확할수록 그 가중치 $\alpha_j$ 는 더 높아지고, 부정확할수록 작아지거나 음수가 된다**.

> *PDF의 주석 그대로*: 랜덤으로 예측해 오류율이 0.5에 가까워지면 $\frac{1-r}{r} \approx 1$이 되어 예측기 가중치가 0에 가까워지고, 오류율이 0.5보다 높으면 $\frac{1-r}{r} < 1$이 되어 가중치가 음수가 된다.

---

#### 식 7-3: 샘플의 가중치 업데이트 규칙 (★★ 가장 중요)

$$
w^{(i)} \leftarrow
\begin{cases}
w^{(i)} & \hat{y}_j^{(i)} = y^{(i)} \text{ 일 때 (잘 맞춘 경우)} \\[6pt]
w^{(i)} \cdot \exp(\alpha_j) & \hat{y}_j^{(i)} \neq y^{(i)} \text{ 일 때 (못 맞춘 경우)}
\end{cases}
\quad (i = 1, 2, \dots, m)
$$

이 식이 AdaBoost의 핵심이다. **하나하나 뜯어보자.**

##### (1) 잘 맞춘 샘플 vs. 못 맞춘 샘플의 차이

- **잘 맞춘 샘플** ($\hat{y}_j^{(i)} = y^{(i)}$):  
  → 가중치를 **그대로 유지**한다 ($w^{(i)} \leftarrow w^{(i)}$).  
  → "이미 잘 맞추니까 더 이상 신경 쓸 필요가 없다."

- **못 맞춘 샘플** ($\hat{y}_j^{(i)} \neq y^{(i)}$):  
  → 가중치에 $\exp(\alpha_j)$ 를 **곱해서 키운다**.  
  → "이 샘플은 어렵다. 다음 예측기는 여기에 더 집중해라."

##### (2) 왜 하필 $\exp(\alpha_j)$ 인가? — **지수적으로 커지는 이유**

$\alpha_j$ 의 부호와 크기에 따라 $\exp(\alpha_j)$ 가 어떻게 동작하는지 보자.

| 예측기 상태 | $\alpha_j$ | $\exp(\alpha_j)$ | 못 맞춘 샘플의 가중치 변화 |
| --- | --- | --- | --- |
| **매우 정확한 예측기** ($r_j$ 작음) | 큰 양수 | **매우 큰 값** ($\gg 1$) | 가중치가 **크게 증가** |
| 그저 그런 예측기 ($r_j \approx 0.5$) | $\approx 0$ | $\exp(0) = 1$ | 가중치 변화 거의 없음 |
| 형편없는 예측기 ($r_j > 0.5$) | 음수 | $< 1$ | 가중치가 오히려 **감소** |

→ **핵심 직관**: "**좋은(정확한) 예측기조차 틀린 샘플은, 진짜로 어려운 샘플이다.**" 그러므로 그런 샘플의 가중치를 **지수적으로 크게** 키워서 다음 예측기가 반드시 맞추도록 강제한다.

##### (3) 언제 가중치가 지수적으로 커지는가?

- 가중치가 **지수적으로(exponential) 커지는 시점**은 **정확한 예측기($\alpha_j$ 가 큰 양수)가 어떤 샘플을 틀렸을 때**이다.
- 반대로, 형편없는 예측기가 틀린 샘플은 별로 신경 쓰지 않는다 ($\exp(\alpha_j) < 1$ 이면 오히려 가중치 감소).
- 즉, AdaBoost는 단순히 "틀린 샘플의 가중치를 키운다"가 아니라, **"신뢰할 만한 예측기가 틀렸다고 판단한 샘플일수록 더 강하게 키운다"** 는 전략이다.

##### (4) 두 식(식 7-2와 식 7-3)의 관계 — **★ 차이점 정리 ★**

학생들이 자주 헷갈리는 부분이다. 두 식은 **이름은 둘 다 "가중치"지만, 서로 완전히 다른 대상에 대한 가중치**다.

| 구분 | 식 7-2: $\alpha_j$ (예측기 가중치) | 식 7-3: $w^{(i)}$ (샘플 가중치) |
| --- | --- | --- |
| **무엇의 가중치인가?** | **j번째 예측기**의 가중치 | **i번째 훈련 샘플**의 가중치 |
| **개수** | 예측기 수 N개 | 훈련 샘플 수 m개 |
| **언제 사용?** | 최종 예측 시(식 7-4)에서 각 예측기의 표에 비중을 매길 때 | **다음 예측기를 훈련할 때** 어려운 샘플을 더 강조하기 위해 |
| **계산 입력** | 오류율 $r_j$ | 이전 가중치 $w^{(i)}$ 와 $\alpha_j$ |
| **방향** | 정확한 예측기 → 큰 값 | 어려운 샘플 → 큰 값 |
| **연결 관계** | $\alpha_j$ 가 큰 양수 → $\exp(\alpha_j)$ 도 큼 → **틀린 샘플의 $w^{(i)}$ 가 크게 커짐** | |

→ 한 줄 요약: **$\alpha_j$ 는 "이 예측기를 얼마나 믿을지"의 가중치이고, $w^{(i)}$ 는 "이 샘플에 얼마나 집중할지"의 가중치다.** 그리고 두 가중치는 **$\exp(\alpha_j)$ 를 통해 연결**된다.

##### (5) 정규화

가중치 업데이트 후, **모든 샘플의 가중치를 정규화**한다 (즉 $\sum_{i=1}^{m} w^{(i)}$ 로 나눈다). 가중치 합이 1이 되도록 만들어야 식 7-1의 오류율 계산이 일관적으로 작동한다.

---

#### 식 7-4: AdaBoost 예측

$$
\hat{y}(\mathbf{x}) \;=\; \underset{k}{\underset{k}{\arg\max}} \sum_{j\,:\,\hat{y}_j(\mathbf{x}) = k}^{} \alpha_j
$$

여기서 $N$은 예측기 수.

**의미**:
- 새로운 샘플 $\mathbf{x}$에 대해 모든 예측기가 클래스 예측을 내놓는다.
- **각 클래스 $k$에 대해 그 클래스를 예측한 예측기들의 $\alpha_j$ 합을 구하고**, 그 합이 가장 큰 클래스를 최종 예측으로 선택한다.
- 즉, 단순 다수결이 아니라 **"신뢰도 높은 예측기에 더 많은 표를 주는 가중 다수결"**.

---

### AdaBoost 알고리즘 전체 흐름 (요약)

1. 모든 샘플에 동일한 초기 가중치 $w^{(i)} = \frac{1}{m}$ 부여
2. 가중치 적용 훈련 세트로 예측기 $j$ 훈련
3. **식 7-1**로 오류율 $r_j$ 계산
4. **식 7-2**로 예측기 가중치 $\alpha_j$ 계산
5. **식 7-3**으로 샘플 가중치 업데이트 (못 맞춘 샘플만 $\exp(\alpha_j)$ 곱셈)
6. 가중치 정규화
7. 다음 예측기 훈련, 2~6 반복
8. 최종 예측은 **식 7-4**로 가중 다수결

### 7.5.2 그레이디언트 부스팅 (Gradient Boosting)

- AdaBoost처럼 **앙상블에 이전까지의 오차를 보정하도록 예측기를 순차적으로 추가**한다.
- **차이점**: AdaBoost는 매 반복마다 **샘플 가중치를 수정**하지만, 그레이디언트 부스팅은 **이전 예측기가 만든 잔여 오차(residual error)에 새로운 예측기를 학습**시킨다.

### 그레이디언트 트리 부스팅 / GBRT

- 결정 트리를 기반 예측기로 사용하는 회귀 문제에서는 이를 **그레이디언트 트리 부스팅(gradient tree boosting)** 또는 **GBRT(gradient boosted regression tree)** 라고 부른다.

#### 학습 과정 (회귀 예시)

1. 첫 번째 트리 $h_1$ 을 원본 데이터 $(X, y)$ 에 학습.
2. 잔여 오차 계산: $y_2 = y - h_1(X)$
3. 두 번째 트리 $h_2$ 를 $(X, y_2)$ 에 학습.
4. 잔여 오차 계산: $y_3 = y_2 - h_2(X)$
5. 세 번째 트리 $h_3$ 를 $(X, y_3)$ 에 학습.
6. 새로운 샘플에 대한 예측은 모든 트리 예측의 합: $h(\mathbf{x}) = h_1(\mathbf{x}) + h_2(\mathbf{x}) + h_3(\mathbf{x})$

#### 그림 7-9 해석 (그레이디언트 부스팅 진행 과정)

- **왼쪽 열 (개별 트리의 예측)**:
  - 위: 첫 번째 트리 $h_1$ 이 원본 데이터에 학습한 예측 (계단형)
  - 중간: 두 번째 트리 $h_2$ 가 첫 트리의 잔여 오차에 학습한 예측
  - 아래: 세 번째 트리 $h_3$ 가 두 번째 트리까지의 잔여 오차에 학습한 예측
- **오른쪽 열 (앙상블 누적 예측)**:
  - 위: $h(\mathbf{x}) = h_1(\mathbf{x})$ — 첫 트리만 사용
  - 중간: $h(\mathbf{x}) = h_1 + h_2$ — 두 트리 합
  - 아래: $h(\mathbf{x}) = h_1 + h_2 + h_3$ — 세 트리 합
- **트리를 더할수록 예측이 점점 데이터를 더 잘 맞추는** 모습이 보인다.

### 축소 (Shrinkage)

- `learning_rate` 매개변수가 **각 트리의 기여도를 조절**한다.
- 이를 0.05처럼 낮게 설정하면, 앙상블이 훈련 세트에 학습되기 위해 더 많은 트리가 필요하지만 **예측 성능은 좋아진다** (일반화 효과).

#### 그림 7-10 해석 (예측기 부족/충분 비교)

- **왼쪽 (`learning_rate=1.0`, `n_estimators=3`)**: 트리 3개로는 데이터를 잘 표현하지 못해 **과소적합**. 결정 경계가 매우 거칠다.
- **오른쪽 (`learning_rate=0.1`, `n_estimators=200`)**: 학습률은 낮지만 트리가 200개여서 데이터를 부드럽게 잘 따라간다.
- → **"낮은 학습률 + 많은 트리"** 조합이 일반적으로 더 좋은 일반화 성능을 낸다.

### 조기 종료와 확률적 그레이디언트 부스팅

- `n_iter_no_change` 하이퍼파라미터를 정수(예: 10)로 설정하면, **마지막 10개 트리가 도움이 되지 않을 경우 자동으로 트리 추가를 중지** (조기 종료).
- 학습된 앙상블의 실제 추정기 개수는 조기 종료 덕분에 훨씬 적을 수 있다 (예: 500개 설정해도 92개에서 멈춤).
- **확률적 그레이디언트 부스팅 (stochastic gradient boosting)**: `subsample` 매개변수로 각 트리가 사용할 훈련 샘플 비율을 지정 (예: `subsample=0.25`이면 25% 샘플만 사용). 편향은 늘지만 분산이 줄고 훈련 속도가 빨라진다.

### 7.5.3 히스토그램 기반 그레이디언트 부스팅 (HGB)

- **HGB (histogram-based gradient boosting)**: 입력 특성을 **구간(bin)으로 나누어 정수로 대체**하는 방식.
- `max_bins` 하이퍼파라미터로 구간 개수 제어 (기본값 255, 더 높게 설정 불가).
- **대규모 데이터셋에서 일반 GBRT보다 수백 배 빠르게 훈련** 가능.
- 단점: **구간 분할은 규제처럼 작동해 정밀도 손실이 발생**할 수 있다.

#### 클래스: `HistGradientBoostingRegressor`, `HistGradientBoostingClassifier`

- 인스턴스 수가 10,000개를 초과하면 조기 종료 자동 활성화 (`early_stopping`로 강제 on/off 가능)
- `subsample` 매개변수 미지원
- `n_estimators` 대신 `max_iter` 사용
- 조정 가능한 결정 트리 하이퍼파라미터: `max_leaf_nodes`, `min_samples_leaf`, `max_depth`만

#### 추가 기능

- **범주형 특성 지원** (단, 0 ~ `max_bins` 범위의 정수로 표현되어야 하므로 `OrdinalEncoder` 사용)
- **누락된 값(missing value) 지원**

#### 캘리포니아 주택 데이터셋 결과 (참고)

- 하이퍼파라미터 튜닝 없이도 RMSE 약 47,613을 기록 (10-fold cross-validation 평균).

## 7.6 스태킹 (Stacking)

### 개념

- **스태킹** (stacked generalization의 줄임말): 앙상블의 예측을 단순한 함수(투표, 평균)로 취합하는 대신, **취합하는 모델 자체를 훈련**시키는 방법.
- 마지막 예측기가 각 모델의 예측을 입력으로 받아 최종 예측을 만든다.
- 이 마지막 예측기를 **블렌더(blender)** 또는 **메타 학습기(meta learner)** 라고 한다.

#### 그림 7-11 해석

- 새로운 샘플이 들어오면, 1층의 세 예측기가 각각 3.1, 2.7, 2.9를 예측한다.
- 이 세 값을 입력으로 받아 블렌더(2층)가 최종 예측 3.0을 출력한다.

### 블렌더 훈련 방법

1. **블렌딩 훈련 세트 만들기**:
   - 앙상블의 모든 예측기에서 `cross_val_predict()`를 사용하여 원본 훈련 세트의 각 샘플에 대한 **표본 외(out-of-fold) 예측**을 얻는다.
   - 이 예측값들이 블렌더 입력 특성이 되고, 타깃은 원본 훈련 세트에서 그대로 복사한다.
2. **블렌더가 학습되면**, 기본 예측기들은 전체 원본 훈련 세트로 마지막에 한 번 더 재훈련된다.

#### 그림 7-12 해석 (블렌더 훈련 구조)

- 훈련 세트 → 1층 예측기들 → 교차 검증 예측(out-of-fold) → 블렌딩 훈련 세트 구성
- 타깃은 훈련 세트에서 직접 복사하여 블렌딩 훈련 세트에 넣음
- 이 블렌딩 훈련 세트로 블렌더(상위 예측기)를 훈련

### 다층 스태킹 (Multi-layer Stacking)

- 여러 가지 블렌더(선형 회귀 블렌더, 랜덤 포레스트 블렌더 등)를 훈련하여 **전체 블렌더 계층을 만든 다음, 그 위에 또 다른 블렌더를 추가**하여 최종 예측 생성.
- 성능을 조금 올릴 수는 있지만 **훈련 시간과 시스템 복잡성 측면에서 비용이 증가**한다.

#### 그림 7-13 해석 (다층 스태킹)

- 층 1: 3개의 기본 예측기 (3.1, 2.7, 2.9 출력)
- 층 2: 3개의 블렌더 (각각 2.9, 2.9, 3.0 출력)
- 층 3: 최종 블렌더 (2.9 출력)
- 모든 층 1 예측기의 출력이 모든 층 2 블렌더의 입력으로 연결되며 (교차 연결), 그 결과가 다시 층 3으로 들어간다.

### 사이킷런의 스태킹 클래스

- `StackingClassifier`, `StackingRegressor`
- moons 데이터셋 예시: 간접 투표 분류기 92%였던 데에서 스태킹은 92.8% — 약간 더 나은 결과.

### 실습 문제 (8~9)

**문제 8: MNIST 앙상블**
- MNIST 데이터를 훈련(50,000) / 검증(10,000) / 테스트(10,000) 세트로 나눔
- 랜덤 포레스트, 엑스트라 트리, SVM, MLP 등 다양한 분류기 훈련
- 직접/간접 투표로 앙상블 → 검증 성능 확인 → 테스트 성능 비교

**문제 9: 스태킹 적용**
- 8번에서 만든 분류기들의 검증 세트 예측을 새로운 훈련 세트로 사용
- 그 위에 블렌더(메타 학습기)를 훈련
- 테스트 세트에 대해 투표 분류기와 비교

#### 강의자료 8번 결과 요약

| 모델 | 검증 정확도 | 테스트 정확도 |
| --- | --- | --- |
| 랜덤 포레스트 | 0.9736 | 0.9680 |
| 엑스트라 트리 | 0.9743 | 0.9703 |
| LinearSVC | 0.8662 | 0.8612 |
| MLP | 0.9659 | 0.9625 |
| **VotingClassifier (4개)** | **0.9752** | **0.9712** |
| **VotingClassifier (SVC 제외)** | **0.9764** | **0.9729** |

→ 가장 약한 분류기(SVC)를 제외하고 앙상블을 다시 만들면 성능이 약간 더 향상된다.

## 정리: 앙상블 방법별 한눈에 비교

| 방법 | 핵심 아이디어 | 병렬화 | 특징 |
| --- | --- | --- | --- |
| **투표 분류기** | 서로 다른 알고리즘의 분류기 결과를 다수결/평균 | ○ | 직접/간접 투표 |
| **배깅/페이스팅** | 같은 알고리즘 + 랜덤 서브셋 | ○ | 분산 감소 |
| **랜덤 포레스트** | 결정 트리 + 배깅 + 특성 무작위성 | ○ | 특성 중요도 측정 가능 |
| **엑스트라 트리** | 랜덤 포레스트 + 임곗값까지 무작위 | ○ | 더 빠름, 편향↑·분산↓ |
| **AdaBoost** | 못 맞춘 샘플 가중치 ↑ + 정확한 예측기 가중치 ↑ | ✗ (순차) | 샘플 가중치 조정 |
| **그레이디언트 부스팅** | 잔여 오차에 새 예측기 학습 | ✗ (순차) | 학습률 조정 가능 |
| **HGB** | GBRT + 히스토그램 구간화 | ✗ (순차) | 대규모에서 매우 빠름 |
| **스태킹** | 블렌더(메타 학습기)가 예측을 합침 | 부분적 | 다층 가능, 복잡함 |
