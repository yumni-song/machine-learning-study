## 📌 이 장에서 다루는 비지도 학습 알고리즘

비지도 학습은 **레이블이 없는 데이터**에서 패턴을 찾는 학습 방식이다. 이 장에서는 크게 세 가지 분야를 다룬다.

### 1. 군집 (Clustering)
- 비슷한 샘플끼리 묶어 **클러스터(cluster)** 로 모으는 작업
- 활용 예: 데이터 분석, 고객 분류, 추천 시스템, 검색 엔진, 이미지 분할, 준지도 학습, 차원 축소 등

### 2. 이상치 탐지 (Outlier Detection)
- '정상' 데이터가 어떻게 보이는지를 학습한 뒤 비정상 샘플(이상치)을 탐지
- 활용 예: 제조 라인에서 결함 제품 감지, 시계열 데이터의 새로운 트렌드 발견

### 3. 밀도 추정 (Density Estimation)
- 데이터셋을 생성한 **확률 과정(random process)** 의 확률 밀도 함수(PDF)를 추정
- 밀도가 매우 낮은 영역에 놓인 샘플 → **이상치일 가능성이 높음**
- 데이터 분석과 시각화에도 유용

## 9.1 군집 (Clustering)

### 분류(Classification) vs. 군집(Clustering)
- **그림 9-1** (붓꽃 데이터셋 예시): 분류는 레이블이 있는 데이터를 학습해 클래스를 예측. 군집은 레이블 없이 데이터의 구조만 보고 비슷한 샘플들을 묶음.
- 같은 데이터라도 분류는 **꽃잎 길이/너비 + 종(Setosa, Versicolor, Virginica) 정보**를 사용하고, 군집은 분포 형태만 가지고 그룹을 찾는다.

군집을 사용하는 다양한 응용 분야:
- 고객 분류, 데이터 분석, 차원 축소 기법, 특성 공학, 이상치 탐지, 준지도 학습, 검색 엔진, 이미지 분할 등

### 9.1.1 k-평균 (k-Means)

**k-평균**은 반복 몇 번만으로 데이터셋을 빠르고 효율적으로 클러스터로 묶을 수 있는 간단한 알고리즘이다. **로이드–포지(Lloyd–Forgy) 알고리즘**이라고도 부른다.

#### 알고리즘 기본 동작
1. 각 클러스터의 **중심(centroid)** 을 찾는다.
2. 각 샘플을 **가장 가까운 센트로이드**가 속한 클러스터에 할당한다.
3. 센트로이드를 재계산하고 다시 할당하는 과정을 반복한다.

훈련된 KMeans 인스턴스는:
- `labels_`: 훈련된 샘플의 예측 레이블
- `cluster_centers_`: 알고리즘이 찾은 센트로이드 위치
- `predict(X_new)`: 새로운 샘플에 가장 가까운 센트로이드의 클러스터를 할당

#### 결정 경계 (보로노이 다이어그램, Voronoi Tessellation)
- **그림 9-3**: 평면을 "특정 점까지의 거리가 가장 가까운 점들의 집합"으로 분할한 그림. 각 영역은 하나의 클러스터에 해당하고, 센트로이드는 × 표시.
- **중요한 한계**: k-평균은 샘플을 클러스터에 할당할 때 **센트로이드까지의 거리만** 고려한다. 따라서 클러스터의 크기·밀집도가 서로 많이 다르면 잘 작동하지 않는다.

#### 하드 군집 vs. 소프트 군집
| 구분 | 설명 |
|------|------|
| **하드 군집(hard clustering)** | 샘플을 정확히 하나의 클러스터에 할당 |
| **소프트 군집(soft clustering)** | 각 클러스터에 대한 점수(예: 거리, 확률 등)를 부여 |

- `KMeans.transform(X)` → 각 샘플과 모든 센트로이드 사이의 **거리**를 반환 (소프트 군집의 일종으로 볼 수 있음).

#### k-평균 알고리즘 상세
1. 센트로이드를 **랜덤하게 초기화** (예: 데이터셋에서 k개를 균등하게 추출).
2. 모든 샘플에 가장 가까운 센트로이드를 할당.
3. 각 클러스터의 평균으로 센트로이드 업데이트.
4. 센트로이드에 변화가 없을 때까지 2~3을 반복.

**수렴 보장**:
- 각 단계마다 "샘플과 가장 가까운 센트로이드 사이의 평균 제곱 거리"는 **감소만 가능**하고 음수가 될 수 없다 → 따라서 알고리즘은 반드시 **유한한 횟수 안에 수렴**한다 (보통 매우 빠르게).

- **그림 9-4**: 랜덤 초기화 후 단 **3번의 반복**만에 최적으로 보이는 클러스터에 도달하는 모습.

#### 지역 최적점 문제
- 수렴은 보장되지만, **반드시 전역 최적점에 도달하지는 않는다** → 지역 최적점(local optimum)에 빠질 수 있음.
- 결과는 **센트로이드 초기화에 크게 의존**한다.
- **그림 9-5**: 같은 데이터에 운 나쁜 초기화를 적용했을 때 만들어지는 부적절한 솔루션 두 가지.

### 센트로이드 초기화 방법

#### 방법 1: 사전 지식으로 초기화 위치 지정
- 다른 군집 알고리즘을 먼저 실행하는 등으로 센트로이드 위치를 근사하게 알 수 있을 때:
  - `init` 매개변수에 센트로이드 리스트(넘파이 배열)를 지정
  - `n_init=1`로 설정

#### 방법 2: 랜덤 초기화를 여러 번 시도
- `n_init` 매개변수로 시도 횟수 조절
- 그중 **가장 좋은 솔루션** 선택
- '좋음'의 기준 → **이너셔(inertia)**

#### 이너셔 (Inertia)
- 정의: 각 샘플과 가장 가까운 센트로이드 사이의 **제곱 거리의 합**
- `kmeans.inertia_`로 확인 가능
- **작을수록 좋음**

| 모델 | 이너셔 |
|------|--------|
| 그림 9-3 (좋은 모델) | 211.6 |
| 그림 9-5 왼쪽 | 219.4 |
| 그림 9-5 오른쪽 | 258.6 |

#### score() 메서드
- `score(X)`는 **이너셔의 음숫값**을 반환
- 사이킷런 관례상 "score 값은 클수록 좋음"을 따르기 위함

### k-평균++ 초기화 알고리즘

기존 랜덤 초기화의 단점(지역 최적점 빠지기 쉬움)을 보완하는 **똑똑한 초기화** 방법. 다른 센트로이드와 거리가 먼 위치를 선택하여 최적 솔루션 수렴까지의 반복 횟수를 크게 줄인다.

#### 동작 방식
1. 데이터셋에서 랜덤으로 균등하게 하나의 센트로이드 $\mathbf{c}^{(1)}$ 선택
2. 다음 확률로 다음 센트로이드 $\mathbf{c}^{(i)}$를 샘플 $\mathbf{x}^{(i)}$ 중에서 선택:

$$
P(\mathbf{x}^{(i)}) = \frac{D(\mathbf{x}^{(i)})^2}{\sum_{j=1}^{m} D(\mathbf{x}^{(j)})^2}
$$

- $D(\mathbf{x}^{(i)})$: 샘플 $\mathbf{x}^{(i)}$와 **이미 선택된 가장 가까운 센트로이드까지의 거리**
- 이 확률 분포는 **이미 선택된 센트로이드에서 멀리 떨어진 샘플**이 다음 센트로이드로 뽑힐 가능성을 높임

3. k개의 센트로이드가 모두 선택될 때까지 2를 반복

→ 사이킷런 `KMeans` 클래스는 **기본적으로 이 방식을 사용**한다.

### k-평균 속도 개선과 미니배치 k-평균

- **MiniBatchKMeans**: 전체 데이터셋을 사용하지 않고 매 반복마다 **미니배치**로 센트로이드를 조금씩 이동.
- **그림 9-6 해석**:
  - 왼쪽 그래프 — 미니배치 k-평균의 이너셔는 일반 k-평균보다 약간 **높다(나쁘다)**.
  - 오른쪽 그래프 — 하지만 훨씬 **빠르다**. 그리고 클러스터 개수 k가 증가할수록 속도 차이는 더 벌어진다.

→ 데이터셋이 크거나 빠른 결과가 필요할 때 유용.

### 최적의 클러스터 개수 찾기

#### 문제점
- 일반적으로 k를 사전에 알기 어렵다.
- 잘못된 k를 고르면 나쁜 모델이 된다.
- **그림 9-7**: k가 너무 작으면(왼쪽, k=3) **별개 클러스터가 합쳐지고**, k가 너무 크면(오른쪽, k=8) **하나의 클러스터가 여러 개로 쪼개진다**.

#### 방법 1: 이너셔만 사용? ❌
- **그림 9-8 해석**: 이너셔는 k가 증가함에 따라 **계속 감소**한다(k=m일 때 0). 따라서 단순히 "가장 작은 이너셔"는 k 선택 기준으로 부적절.
- **엘보우(elbow) 기법**: 이너셔 감소율이 급격히 꺾이는 지점(팔꿈치)을 고르는 휴리스틱. 그림 9-8에서는 **k=4 부근**.
- 단점: 다소 주관적이고 정확하지 않다.

#### 방법 2: 실루엣 점수 (Silhouette Score)
- **모든 샘플에 대한 실루엣 계수(silhouette coefficient)의 평균**.

##### 샘플의 실루엣 계수 정의
$$
s = \frac{b - a}{\max(a, b)}
$$

- $a$ = **동일한 클러스터에 있는 다른 샘플까지의 평균 거리** (클러스터 내부 평균 거리)
- $b$ = **가장 가까운 클러스터까지의 평균 거리**
  - "가장 가까운 클러스터" = 자신이 속한 클러스터를 제외하고 $b$값이 최소가 되는 클러스터
  - 그 클러스터의 모든 샘플까지의 평균 거리

##### 실루엣 계수의 의미 (범위: -1 ~ +1)
| 값 | 의미 |
|----|------|
| **+1에 가까움** | 자신의 클러스터 안에 잘 속해 있고 다른 클러스터와는 멀리 떨어짐 (이상적) |
| **0에 가까움** | 클러스터 **경계**에 위치 |
| **-1에 가까움** | **잘못된 클러스터**에 할당됨 |

- **그림 9-9 해석**: k=2부터 8까지의 실루엣 점수를 비교하면 **k=4에서 가장 높은 점수(약 0.69)**, 그다음이 k=5. → 엘보우 그래프만 봤을 때보다 더 명확한 정보를 준다.

#### 방법 3: 실루엣 다이어그램 (Silhouette Diagram)
- 모든 샘플의 실루엣 계수를 **할당된 클러스터별로, 그리고 계수 값으로 정렬**하여 그린 그래프.
- 각 클러스터마다 칼날(knife) 모양이 형성됨.

##### 해석 방법
| 요소 | 의미 |
|------|------|
| **높이** | 클러스터가 포함하고 있는 샘플의 개수 |
| **너비** | 클러스터에 속한 샘플의 실루엣 계수 (**넓을수록 좋음**) |
| **수직 파선** | 해당 k에 대한 **평균 실루엣 점수** |

- **그림 9-10 해석** (k = 3, 4, 5, 6):
  - 평균 실루엣 점수가 높은 것뿐 아니라 **각 클러스터의 너비가 평균선을 넘는지**, **클러스터 크기가 균형 있는지** 함께 봐야 한다.
  - k=4일 때 가장 평균이 높지만, k=5일 때 클러스터들의 크기가 더 균형 있고 모두 평균선을 넘는 경우가 있어 더 좋은 선택일 수 있다.

### 9.1.2 k-평균의 한계

#### 장점
- 속도가 빠르고 확장이 용이하다.

#### 단점
1. 최적이 아닌 솔루션(지역 최적)을 피하려면 **알고리즘을 여러 번 실행**해야 한다.
2. **클러스터 개수 k를 사전 지정**해야 한다.
3. **클러스터의 크기·밀집도가 서로 다르거나, 원형이 아닐 때** 잘 작동하지 않는다.

- **그림 9-11 해석**: 세 개의 **타원형 클러스터**를 가진 데이터에서 k-평균은 적절히 구분하지 못한다. 두 솔루션이 모두 타원의 형태와 방향을 무시한 채 거리 기반으로만 분할하므로 자연스럽게 잘못된 경계가 생긴다. 이너셔 값은 솔루션 간 작은 차이를 보이지만 둘 다 옳지 않다.

### 9.1.3 군집을 사용한 이미지 분할

#### 이미지 분할(image segmentation)의 종류
| 분할 종류 | 설명 | 예시 |
|----------|------|------|
| **색상 분할** | 동일한 색상을 가진 픽셀을 같은 세그먼트에 할당 | 인공위성 사진으로 한 지역의 산림 면적 측정 |
| **시맨틱 분할** | 동일한 **종류의 물체**에 속한 픽셀을 같은 세그먼트에 할당 | 자율주행 차량의 비전 시스템에서 모든 '보행자' 픽셀을 한 세그먼트로 |
| **인스턴스 분할** | **개별 객체**에 속한 픽셀을 같은 세그먼트에 할당 | 보행자 A, 보행자 B를 각각 다른 세그먼트로 |

→ k-평균은 매우 간단한 **색상 분할**에 사용 가능 (시맨틱/인스턴스 분할에는 보통 딥러닝 모델이 필요).

#### k-평균을 사용한 색상 분할 방법
1. 이미지를 RGB 색상 리스트로 reshape (예: (533, 800, 3) → (533·800, 3))
2. k-평균을 적용해 색상을 k개의 클러스터로 묶음
3. 각 픽셀의 색을 자신이 속한 **클러스터의 센트로이드 색**으로 대체
4. 다시 원본 이미지 형태로 reshape

- **그림 9-12 해석**: 무당벌레(ladybug) 사진을 k = 10, 8, 6, 4, 2로 분할한 결과.
  - 클러스터가 많을수록 원본에 가깝지만, 적어질수록 색이 단순화된다.
  - **k=2**에서는 빨간색 무당벌레가 사라지는 것에 주목 → 무당벌레 빨간색이 차지하는 픽셀 수가 적어, "가까운 클러스터"가 노란색 또는 녹색으로 흡수된 것. k-평균이 단순히 픽셀 수와 거리에 따라 결정함을 보여준다.

### 9.1.4 군집을 사용한 준지도 학습

**준지도 학습(semi-supervised learning)**: 레이블 없는 데이터는 많고, 레이블 있는 데이터는 적은 상황에서 사용.

#### 실험 시나리오 (8×8 흑백 숫자 이미지 1,797개, MNIST 유사)
1. 훈련 세트 1,400개 중 **50개에만 레이블이 있다고 가정** → 로지스틱 회귀 훈련.
   - **정확도 약 74.8%**
2. 군집을 활용한 개선 단계:
   - 훈련 세트를 **k=50 클러스터로 군집**
   - 각 클러스터에서 **센트로이드에 가장 가까운 이미지 = 대표 이미지** 선택 (50개)
   - **그림 9-13**: 50개의 대표 숫자 이미지 (클러스터당 1개) — 사람이 보고 손으로 레이블을 부여
3. 이 50개 대표 이미지로만 다시 훈련 → **정확도 약 88.7%** (대폭 향상!)

#### 레이블 전파 (Label Propagation)
- 대표 이미지에 부여한 레이블을 **같은 클러스터의 모든 샘플**에 전파한다.
- 50개 → 1,400개로 레이블 확장 후 재훈련 → **정확도 약 89.4%**

#### 이상치 제거 + 레이블 전파
- 클러스터 중심에서 가장 먼 **1%의 샘플을 무시**(이상치로 간주)하여 부분적인 전파만 수행.
- 결과:
  - **분류 정확도: 약 90.9%**
  - **전파된 레이블의 정확도: 약 97.5%**
- 즉, 평균 클래스당 5개 샘플만 사람이 직접 레이블링했을 뿐인데도 매우 높은 정확도를 얻을 수 있다.

#### 사이킷런의 자동 전파 클래스
- `sklearn.semi_supervised` 패키지의 `LabelSpreading`, `LabelPropagation` 클래스가 레이블 자동 전파를 제공한다.

### 능동 학습 (Active Learning)

준지도 학습을 더 발전시킨 방법. **전문가가 학습 알고리즘과 상호작용**하여 알고리즘이 요청할 때 특정 샘플의 레이블을 제공.

#### 가장 널리 쓰이는 전략: 불확실성 샘플링 (Uncertainty Sampling)
1. 지금까지 수집한 레이블된 샘플로 모델 훈련 → 레이블 없는 모든 샘플에 예측을 만듦
2. **모델이 가장 불확실하게 예측한 샘플**(즉, 추정 확률이 가장 낮은 샘플)을 전문가에게 보내 레이블 부여
3. 레이블링 노력 대비 성능 향상이 거의 없어질 때까지 반복

#### 다른 능동 학습 전략들
- 모델을 **가장 크게 바꾸는** 샘플에 대해 레이블 요청
- 모델의 **검증 점수를 가장 크게 떨어뜨리는** 샘플
- 여러 개의 모델(예: SVM, 랜덤 포레스트)이 **동일하지 않은 예측**을 내는 샘플

### 9.1.5 DBSCAN

**DBSCAN** (Density-Based Spatial Clustering of Applications with Noise): **밀집된 연속적 지역**을 클러스터로 정의.

#### 작동 방식
1. 각 샘플에서 작은 거리 $\varepsilon$ 내에 샘플이 몇 개 있는지 계산. 이 지역을 샘플의 **$\varepsilon$-이웃($\varepsilon$-neighborhood)** 이라고 함.
2. 자기 자신을 포함하여 $\varepsilon$-이웃 내에 적어도 `min_samples` 개의 샘플이 있다면 그 샘플을 **핵심 샘플(core instance)** 로 간주.
   - 핵심 샘플 = **밀집된 지역에 있는 샘플**
3. 핵심 샘플의 이웃에 있는 모든 샘플은 **동일한 클러스터**에 속함.
   - 그 이웃에 또 다른 핵심 샘플이 있다면 그 핵심 샘플의 이웃까지 같은 클러스터로 확장된다.
   - → **이웃의 이웃**으로 계속 연결되며 하나의 클러스터를 형성.
4. **핵심 샘플도 아니고, 핵심 샘플의 이웃도 아닌 샘플**은 **이상치(노이즈)** 로 판단.

#### DBSCAN의 주요 속성
| 속성 | 설명 |
|------|------|
| `labels_` | 모든 샘플의 클러스터 인덱스. **-1은 이상치**를 의미 |
| `core_sample_indices_` | 핵심 샘플의 인덱스 |
| `components_` | 핵심 샘플 그 자체(원본 좌표) |

#### 그림 9-14 해석
- **왼쪽** (`eps=0.05`, `min_samples=5`): 클러스터가 7개나 만들어졌고, 많은 샘플이 이상치(빨간 X)로 판단됨. → 이웃 반경이 너무 작아 같은 형태의 곡선조차 끊긴다.
- **오른쪽** (`eps=0.20`, `min_samples=5`): 이웃 반경을 넓혔더니 **두 개의 반달 모양 클러스터를 완벽히 식별**.
- 결론: DBSCAN의 결과는 **`eps` 값에 매우 민감**.

#### 예측은 별도의 분류기로
- `DBSCAN` 클래스는 `predict()`를 **제공하지 않는다** (오직 `fit_predict()`).
- 즉, **새로운 샘플에 대한 클러스터를 직접 예측할 수 없음**.
- 해결책: 사용자가 원하는 분류기(예: `KNeighborsClassifier`)를 핵심 샘플 위에 훈련시켜 사용.

#### KNN을 통한 예측 + 이상치 분류 (그림 9-15)
- `KNeighborsClassifier`의 `kneighbors()`로 새 샘플과 가장 가까운 핵심 샘플의 거리를 측정.
- **최대거리(예: 0.2)** 보다 멀면 → **이상치로 분류**.
- 그림 9-15에서 X_new 샘플 4개 중 멀리 떨어진 샘플은 -1(이상치)로 분류, 두 반달 영역에 가까운 샘플은 각각의 클러스터로 분류된다.

#### 학습 대상 선택의 유연성
- 분류기를 **핵심 샘플에만** 훈련시킬 수도 있고
- **모든 샘플**에 훈련시킬 수도 있고
- **이상치를 제외**한 샘플에 훈련시킬 수도 있다
- → **최종 작업의 성능**에 따라 선택한다.

### 9.1.6 다른 군집 알고리즘

| 알고리즘 | 핵심 아이디어 |
|----------|-------------|
| **병합 군집 (Agglomerative)** | 클러스터 계층을 **밑바닥부터 위로** 쌓아 구성 |
| **BIRCH** (balanced iterative reducing and clustering using hierarchies) | 특별히 **대규모 데이터셋**을 위해 고안 |
| **평균-이동 (Mean-Shift)** | 각 샘플 중심으로 원을 그리고, 원 안 샘플의 평균으로 중심을 이동. 모든 원이 움직이지 않을 때까지 반복 |
| **유사도 전파 (Affinity Propagation)** | 샘플 간 메시지를 반복적으로 교환하면서 각 샘플이 자신을 대표할 **예시(exemplar)** 를 선택 |
| **스펙트럼 군집 (Spectral Clustering)** | 샘플 간 **유사도 행렬**을 받아 **저차원 임베딩**(차원 축소) 생성 → 저차원 공간에서 다른 군집 알고리즘 사용 |

## 9.2 가우스 혼합 (Gaussian Mixture)

### 가우스 혼합 모델 (Gaussian Mixture Model, GMM)
- **샘플이 파라미터를 알 수 없는 여러 가우스 분포의 혼합으로부터 생성되었다고 가정**하는 확률 모델.
- 가장 간단한 형태가 `GaussianMixture` 클래스에 구현되어 있으며, 가우스 분포의 개수 $k$를 사전에 지정해야 한다.

### 생성 과정 (Generative Process) 가정
데이터셋 $X$가 다음 확률 과정으로 생성되었다고 가정한다.

각 샘플 $i$에 대해:
1. $k$개의 클러스터 중 하나를 랜덤하게 선택. $j$번째 클러스터가 뽑힐 확률은 **클러스터 가중치** $\phi^{(j)}$.
   - 선택된 클러스터 인덱스를 $z^{(i)}$로 표기.
2. $z^{(i)} = j$라면, 샘플의 위치는 다음 가우스 분포에서 추출:

$$
\mathbf{x}^{(i)} \sim \mathcal{N}\big(\boldsymbol{\mu}^{(j)}, \boldsymbol{\Sigma}^{(j)}\big)
$$

- $\boldsymbol{\mu}^{(j)}$: 클러스터 $j$의 **평균 벡터**
- $\boldsymbol{\Sigma}^{(j)}$: 클러스터 $j$의 **공분산 행렬**

### 파라미터 학습 — EM 알고리즘
주어진 데이터 $X$로부터 다음을 추정한다:
- 가중치 $\phi^{(1)}, \dots, \phi^{(k)}$
- 평균 $\boldsymbol{\mu}^{(1)}, \dots, \boldsymbol{\mu}^{(k)}$
- 공분산 $\boldsymbol{\Sigma}^{(1)}, \dots, \boldsymbol{\Sigma}^{(k)}$

#### 기댓값-최대화 (Expectation-Maximization, EM) 알고리즘
k-평균을 일반화한 알고리즘:
- **E단계 (Expectation step)** — 샘플을 클러스터에 할당 (확률적으로)
- **M단계 (Maximization step)** — 클러스터를 업데이트 (가중치, 평균, 공분산)
- 두 단계를 수렴까지 반복.

#### 책임 (Responsibility)
- 각 클러스터에 속할 **추정 확률**을 통해 샘플에 가중치를 부여하여 클러스터를 업데이트.
- 이 확률을 샘플에 대한 클러스터의 **책임(responsibility)** 이라 부른다.

### 학습된 모델 활용
| 메서드/속성 | 의미 |
|------------|------|
| `gm.weights_` | 학습된 클러스터 가중치 $\phi$ |
| `gm.means_` | 학습된 클러스터 평균 $\boldsymbol{\mu}$ |
| `gm.covariances_` | 학습된 공분산 $\boldsymbol{\Sigma}$ |
| `gm.converged_` | 수렴 여부 |
| `gm.n_iter_` | 반복 횟수 |
| `gm.predict(X)` | **하드 군집** — 가장 비슷한 클러스터 인덱스 |
| `gm.predict_proba(X)` | **소프트 군집** — 각 클러스터에 속할 확률 |
| `gm.sample(n)` | 모델로부터 **새로운 샘플 생성** (생성 모델!) |
| `gm.score_samples(X)` | 주어진 위치에서 PDF의 **로그**를 예측 (밀도 추정) |

#### 생성 모델로서의 GMM
- GMM은 **생성 모델(generative model)** — 학습 후 새로운 샘플을 만들어낼 수 있다.
- `sample()`은 각 클러스터에서 **순차적으로** 샘플링됨을 주의 (예: 클러스터 0에서 먼저, 그다음 1, 2...).

### 그림 9-16 해석
- 훈련된 GMM의 시각화:
  - **× 표시** = 각 클러스터의 평균 (센트로이드)
  - **빨간 파선** = 결정 경계
  - **곡선 등고선** = 모델의 **밀도 등고선** (PDF 값이 같은 점들)
- 세 개의 타원형 클러스터가 잘 잡혀 있다 (k-평균이 실패한 그림 9-11과 비교하면 명확).

---

### 공분산 행렬에 대한 제약 (covariance_type)

특성이나 클러스터가 많거나, 샘플이 적을 때는 EM이 최적 솔루션으로 수렴하기 어렵다. **학습 파라미터의 개수를 줄여** 학습을 안정화한다.

#### `covariance_type` 매개변수 옵션
| 값 | 클러스터 모양 제약 |
|----|--------------------|
| **`"spherical"`** | 모든 클러스터가 **원형**. 지름(분산)은 다를 수 있음. |
| **`"diag"`** | 어떤 타원형도 가능하지만, **타원의 축이 좌표축과 나란해야 함** (공분산 행렬이 대각 행렬). |
| **`"tied"`** | 모든 클러스터가 **동일한 모양·크기·방향**을 가짐 (모든 클러스터가 같은 공분산 행렬 공유). |
| **`"full"`** (기본값) | 제약 없음 — 모든 클러스터가 자유로운 타원형. |

#### 그림 9-17 해석
- **왼쪽 (`tied`)**: 모든 클러스터가 같은 타원 모양·크기·방향. → 결정 경계가 직선 형태.
- **오른쪽 (`spherical`)**: 모든 클러스터가 원형. 지름은 다를 수 있음. → 결정 경계가 방사형으로 단순.

### 9.2.1 가우스 혼합을 사용한 이상치 탐지

#### 기본 아이디어
- 학습된 GMM의 밀도가 **낮은 지역**에 있는 모든 샘플을 **이상치로 간주**.
- 사용자가 적절한 **임곗값**을 직접 정한다.
- 일반적인 방법: **백분위수**를 임곗값으로 사용 (예: 하위 4% → `np.percentile(densities, 4)`).
- 이상치 탐지율(거짓 양성률)과 누락률(거짓 음성률)의 균형을 보고 임곗값을 조정.

#### 그림 9-18 해석
- 별표로 표시된 빨간 점들이 **이상치로 분류된 샘플**.
- 그래프의 등고선 외곽(밀도가 낮은 곳)에 위치한다.

#### 이상치 탐지 vs 특이치 탐지의 차이
- **특이치 탐지(novelty detection)**: 이상치로 **오염되지 않은 '깨끗한' 데이터셋**에서 모델을 훈련. 이후 새로운 샘플이 특이치인지 판단.
- **이상치 탐지(outlier detection)**: 훈련 데이터에 이미 이상치가 섞여 있어도 가능.

### 9.2.2 클러스터 개수 선택

GMM에서는 이너셔나 실루엣 점수가 잘 맞지 않는다. 대신 **이론적 정보 기준**을 최소화하는 모델을 선택.

#### 식 9-1: BIC와 AIC

$$
\boxed{BIC = \log(m) \cdot p - 2 \log\!\big(\hat{\mathcal{L}}\big)}
$$

$$
\boxed{AIC = 2p - 2 \log\!\big(\hat{\mathcal{L}}\big)}
$$

- $m$: **샘플의 개수**
- $p$: 모델이 학습할 **파라미터 개수**
- $\hat{\mathcal{L}}$: 모델의 **가능도 함수(likelihood function)의 최댓값**

#### BIC와 AIC의 의미와 차이
| 기준 | 설명 |
|------|------|
| **BIC** (Bayesian Information Criterion) | 모델 복잡도에 대한 페널티가 $\log(m) \cdot p$ — 샘플 수가 많을수록 페널티가 커진다. **더 단순한 모델을 선호**. |
| **AIC** (Akaike Information Criterion) | 페널티가 $2p$ — 샘플 수와 무관하게 일정. |
| 공통점 | 두 기준 모두 **작을수록 좋다**. 가능도가 높으면(데이터를 잘 설명하면) 좋고, 파라미터가 적으면(단순하면) 좋다. |

#### 그림 9-20 해석
- k = 1부터 9까지의 BIC와 AIC 값 변화.
- **k=3에서 두 값 모두 최소** → 이 데이터셋의 최적 클러스터 수는 3.

### 9.2.3 베이즈 가우스 혼합 모델 (BayesianGaussianMixture)

#### 핵심 아이디어
- 클러스터 개수를 수동으로 찾지 않고, **불필요한 클러스터의 가중치를 0(또는 0에 가깝게)** 만들어 자동으로 제거한다.
- 사용 시 충분히 큰 클러스터 개수(예: 10)를 지정만 하면 됨.

#### 예시
- `BayesianGaussianMixture(n_components=10)`로 학습 → `weights_`를 보면 가중치가 0이 아닌 클러스터는 **3개 정도**로 줄어든다 (예: `[0.4, 0.21, 0.4, 0, 0, 0, 0, 0, 0, 0]`).
- 즉, 알고리즘이 자동으로 "이 데이터는 3개 클러스터로 설명할 수 있다"고 판단.

#### 한계
- 베이즈 가우스 혼합 역시 **타원형 클러스터를 가정**한다.
- **그림 9-21 해석**: 반달(moons) 데이터처럼 **타원형이 아닌 클러스터**에서는 잘 작동하지 않는다. 알고리즘이 곡선 모양을 여러 개의 작은 타원으로 강제로 쪼개버린다.

### 9.2.4 이상치/특이치 탐지를 위한 기타 알고리즘

| 알고리즘 | 특징 / 용도 |
|----------|-------------|
| **Fast-MCD** (Minimum Covariance Determinant) | 사이킷런 `EllipticEnvelope`에서 구현. 이상치 탐지, 특히 **데이터셋 정제**에 유용 |
| **아이솔레이션 포레스트 (Isolation Forest)** | **고차원 데이터셋**에서의 이상치 탐지에 효율적 |
| **LOF** (Local Outlier Factor) | 이상치 탐지에 좋음. **주어진 샘플 주위 밀도와 이웃 주위 밀도를 비교** |
| **one-class SVM** | **특이치 탐지**에 효과적 |
| **PCA (그리고 inverse_transform이 가능한 차원 축소 기법)** | 이상치 탐지에 매우 효과적이며 간단함 (재구성 오차로 판단) |

## 📝 연습문제 10. 올리베티 얼굴 데이터셋

### 데이터셋 소개
- 전통적인 **올리베티(Olivetti) 얼굴 데이터셋**: 64×64 픽셀 흑백 얼굴 이미지 **400개**
- 각 이미지는 **4,096 크기의 1D 벡터**로 펼쳐져 있음
- **40명 × 10장**의 사진으로 구성
- 일반적 작업: 어떤 사람의 사진인지 예측하는 모델 훈련

### (a) 데이터셋 로드 및 분할
- `sklearn.datasets.fetch_olivetti_faces()` 함수로 불러옴 (이미 0~1로 스케일 조정됨)
- 훈련/검증/테스트 세트로 분할
- 데이터셋이 작으므로 **계층적 샘플링(StratifiedShuffleSplit)** 으로 각 세트에 동일 인물이 고루 포함되게 함
- 분할 결과 예시: 훈련 280개, 검증 80개, 테스트 40개

### (b) k-평균으로 군집 및 적절한 k 찾기
- 차원이 4,096으로 매우 높으므로 작업 속도를 위해 **PCA(분산의 99% 보존)** 로 차원 축소 (예: 199차원)
- k = 5, 10, 15, ..., 145의 KMeans 모델 학습
- **실루엣 점수**로 최적 k 선택
- 실루엣 점수 그래프 해석:
  - k가 작을 때(특히 15 부근)는 점수가 낮음
  - k가 커질수록 점수가 점점 올라감
  - **k=120 근처에서 최대값** (40명 × 약 3 클러스터/사람 정도가 적합한 경향)
  - 이너셔로 엘보우 포인트를 찾기는 어려움 (얼굴 데이터는 매끄럽게 감소)

### (c) 클러스터 시각화 — 각 클러스터에 비슷한 얼굴이 들어있나?
- 각 클러스터의 샘플들을 시각화
- 해석 예시:
  - **Cluster 0**: 라벨 28번 인물의 사진 3장 → **같은 사람**
  - **Cluster 1**: 라벨 37번 인물 1장 → 외톨이 클러스터
  - **Cluster 2**: 라벨 22, 24번이 섞임 → **비슷하게 생긴 두 사람**이 같은 클러스터에 묶임
  - **Cluster 3**: 라벨 10번 4장 → 같은 사람, 잘 묶임
  - **Cluster 5**: 라벨 28, 38번이 섞임
- → k-평균은 **얼굴의 시각적 유사도**를 기준으로 묶으므로 한 사람이 여러 클러스터에 나뉘기도 하고, 비슷하게 생긴 다른 사람들이 한 클러스터에 들어가기도 한다.

## 🎯 핵심 요약

1. **비지도 학습 = 레이블 없이 데이터의 구조를 찾는 학습** (군집 / 이상치 탐지 / 밀도 추정).
2. **k-평균**: 빠르고 단순하지만, 클러스터 개수 지정, 초기화 의존성, 비원형 클러스터 취약이라는 한계.
3. **최적 k 선택**: 이너셔(엘보우) → 보조적, **실루엣 점수와 다이어그램이 더 신뢰성 있음**.
4. **DBSCAN**: 밀도 기반, 임의 모양 클러스터 처리 가능, 이상치 자동 식별. `eps`에 민감.
5. **GMM**: 확률 모델로서 **소프트 군집·생성·밀도 추정·이상치 탐지** 모두 지원. EM 알고리즘으로 학습.
6. **BIC/AIC**: GMM에서 최적 클러스터 개수를 자동으로 선택하는 정보 기준.
7. **BayesianGaussianMixture**: 불필요한 클러스터의 가중치를 자동으로 0에 가깝게 만듦.
8. **준지도 학습**: 군집을 통해 적은 레이블로도 큰 성능 향상이 가능 (대표 이미지 → 레이블 전파 → 이상치 제거).
9. **능동 학습**: 모델이 가장 불확실한 샘플에 대해서만 사람의 레이블을 요청 → 효율적인 학습.
