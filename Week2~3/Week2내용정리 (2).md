## 2.5 머신러닝 알고리즘을 위한 데이터 준비

### 📌 데이터 자동화 및 분리
* **장점**: 어떤 데이터셋에도 변환 반복 가능, 재사용 가능한 라이브러리 구축.
* **예측 변수와 레이블 분리**: 훈련 세트에 동일한 변형이 적용되지 않도록 타깃값을 별도로 보관합니다.
```python
housing = strat_train_set.drop("median_house_value", axis=1) # 예측 변수
housing_labels = strat_train_set["median_house_value"].copy() # 타깃 레이블
```

### 2.5.1 데이터 정제 (Data Cleaning)
* **SimpleImputer**: 누락된 값을 중간값(median) 등으로 대체합니다.
* **사이킷런 설계 철학**: 
    * **일관성**: 모든 객체(추정기, 변환기)가 단순한 인터페이스 공유.
    * **검사 가능**: 하이퍼파라미터와 학습된 파라미터에 직접 접근 가능.
    * **조합성**: 기존 구성 요소를 파이프라인으로 연결하여 재사용.

### 2.5.2 텍스트와 범주형 특성 다루기
* **OrdinalEncoder**: 텍스트 카테고리를 숫자로 변환합니다.
* **OneHotEncoder**: 카테고리를 원-핫 벡터(이진 특성)로 변환합니다.
    * 기본값은 **사이파이(SciPy) 희소행렬**을 반환하여 메모리를 절약합니다.
    * `sparse=False` 설정 시 넘파이 배열로 반환됩니다.
* **get_dummies()**: 판다스에서 제공하는 원-핫 인코딩 함수입니다.

### 2.5.3 특성 스케일과 변환
* **스케일링**: 모든 특성의 범위를 일정하게 맞춥니다(정규화/표준화).
* **분포 변환**: 
    * **로그 변환**: 꼬리가 긴 멱법칙 분포를 종 모양으로 변환.
    * **버킷타이징**: 연속형 데이터를 균등한 구간으로 나눔.
    * **가우스 RBF**: 특정 지점과의 거리에 따른 유사도를 측정.
* **역변환**: `inverse_transform()`을 통해 변환된 값을 원래대로 돌립니다.

### 2.5.4 사용자 정의 변환기
* **나만의 변환기**: `fit()`, `transform()`, `fit_transform()` 메서드를 구현하여 파이프라인에 통합합니다.
* **군집 활용**: k-평균(k-means) 알고리즘을 사용해 위치 데이터를 클러스터링하고 이를 새로운 특성으로 추가할 수 있습니다.

### 2.5.5 변환 파이프라인
* **Pipeline**: 여러 변환 단계를 순차적으로 실행합니다.
* **ColumnTransformer**: 열마다 다른 변환기(수치형 vs 범주형)를 적용합니다.
* **결과**: 모든 처리가 완료되면 최종적으로 학습 준비가 된 넘파이 배열이 출력됩니다.

## 2.6 모델 선택과 훈련

### 2.6.1 모델 훈련 및 평가
* **선형 회귀 (Linear Regression)**: 기본적인 회귀 모델.
* **결정 트리 (DecisionTreeRegressor)**: 강력하지만 훈련 데이터에 **과대적합**되기 쉬움.

### 2.6.2 교차 검증 (Cross-Validation)
* **k-폴드 교차 검증**: 데이터를 k개의 서브셋으로 나누어 모델을 반복 검증합니다.
* **랜덤 포레스트 (RandomForestRegressor)**: 여러 결정 트리를 앙상블하여 예측의 평균을 구합니다.

## 2.7 모델 미세 튜닝 (Fine-Tuning)

### 2.7.1 & 2.7.2 그리드 서치와 랜덤 서치
| 구분 | GridSearchCV | RandomizedSearchCV |
| :--- | :--- | :--- |
| **작동 방식** | 지정한 모든 하이퍼파라미터 조합 시도 | 랜덤한 조합을 지정한 횟수만큼 시도 |
| **특징** | 조합이 적을 때 유리, 꼼꼼한 탐색 | 탐색 공간이 클 때 유리, 자원 효율적 |

### 2.7.3 앙상블 방법
* **개념**: 각기 다른 오차를 내는 최상의 모델들을 연결하여 더 나은 성능을 도출합니다.
* **예**: 랜덤 포레스트 + k-최근접 이웃의 예측값을 평균 내어 사용.

### 2.7.4 최상의 모델 분석
* **특성 중요도**: 모델이 어떤 특성을 중요하게 생각하는지 확인하고, 중요도가 낮은 특성을 제거하여 모델을 간소화합니다.

### 2.7.5 테스트 세트로 평가
* 최종 모델을 테스트 세트에 적용하여 **일반화 오차**를 측정합니다. 하이퍼파라미터를 테스트 세트에 맞추려 하면 안 됩니다.
